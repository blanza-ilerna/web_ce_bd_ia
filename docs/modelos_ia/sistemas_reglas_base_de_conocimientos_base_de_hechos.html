Base de Conocimientos y de Hechos

Diferenciando el saber experto de la realidad actual

Para que un sistema basado en reglas pueda razonar, necesita gestionar dos tipos de información: una estática, que contiene el saber experto, y otra dinámica, que describe la situación concreta a resolver.

1. La Base de Conocimientos: El Saber Experto

Es el núcleo del sistema y su parte más estable. Representa la "memoria a largo plazo" y se compone de reglas lógicas que definen las relaciones entre diferentes variables del dominio.

📋 Estructura de una Regla

Cada regla se divide en dos partes fundamentales:

  • Antecedente (SI...): La condición o premisa que debe cumplirse.
  • Consecuente (ENTONCES...): La conclusión o acción que se deriva.

Ejemplo: "SI el paciente tiene tos y fiebre, ENTONCES el posible diagnóstico es gripe".

Un sistema robusto puede albergar miles de estas reglas, codificando décadas de experiencia humana en un formato procesable por la máquina.

2. La Base de Hechos: La Realidad Dinámica

A diferencia de la anterior, la base de hechos es cambiante. Representa la "memoria de trabajo" y contiene los datos específicos del problema actual: síntomas, lecturas de sensores o respuestas del usuario.

🔄 Naturaleza Incremental

La base de hechos no es estática. A medida que el motor de inferencia aplica reglas, se generan nuevos hechos que se añaden a la base, permitiendo un razonamiento progresivo.

📡 Fuentes de Información

Los hechos pueden provenir de entradas manuales de un operador, de registros históricos de una base de datos o de telemetría capturada en tiempo real por hardware.

Simbiosis de Memoria

Si comparamos el sistema con un médico: la Base de Conocimientos es todo lo que aprendió en la carrera y sus años de práctica, mientras que la Base de Hechos es la ficha clínica del paciente que tiene delante en este momento.
