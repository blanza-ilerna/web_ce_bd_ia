Fases de Resolución de un Problema

El camino estructurado desde la definición hasta la ejecución

Resolver un problema mediante Inteligencia Artificial no es un proceso aleatorio, sino una secuencia estructurada que permite transformar una necesidad del mundo real en un conjunto de instrucciones que una máquina puede procesar, resolver y verificar.

1. Formulación del Problema

Es el primer paso crítico donde se define formalmente qué se quiere resolver. Consiste en identificar los elementos fundamentales que componen el entorno del problema.

📐 Definición de Componentes

Se establecen el estado inicial, los objetivos a alcanzar y los operadores (acciones permitidas) que pueden modificar el estado actual.

🔍 Nivel de Abstracción

Es vital determinar el detalle adecuado. En planificación de tráfico, nos interesan las vías principales y flujos, no cada semáforo individual, para evitar una complejidad innecesaria.

2. Representación del Conocimiento

En esta etapa, la realidad se traduce a un modelo interno que el sistema puede manipular algorítmicamente.

📊 Estructuras de Datos

Dependiendo del problema, se utilizan grafos, listas o árboles de decisión. Por ejemplo, para calcular la ruta más corta entre ciudades, se suele emplear un grafo donde los nodos son localidades y las aristas representan las distancias o costes de conexión.

3. Búsqueda de Soluciones

Con el problema modelado, el sistema explora el espacio de estados para encontrar la secuencia de acciones óptima.

🛤️ Exploración del Espacio

El sistema genera nuevos estados aplicando operadores y evalúa mediante algoritmos cuáles le acercan más a la solución deseada.

🎲 Incertidumbre

La búsqueda puede ser determinista (un solo camino correcto) o probabilística si el entorno presenta factores desconocidos o aleatorios.

4. Ejecución y Verificación

La fase final implica aplicar el plan obtenido y validar que los resultados coincidan con las expectativas iniciales.

✅

Retroalimentación

En sistemas avanzados, esta fase incluye procesos de aprendizaje automático, permitiendo que el sistema analice sus propios errores para mejorar su desempeño en ejecuciones futuras.
