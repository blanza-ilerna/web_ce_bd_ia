UD 4. Sistemas de Razonamiento y Planificación

Damos un paso más en la automatización: de las estrategias de búsqueda al razonamiento inteligente.

Tras entender cómo buscar soluciones, ahora aprenderemos cómo las máquinas conectan ideas para justificar decisiones.

¿Qué es el Razonamiento?

🧠 Actividad Mental

Proceso de conectar ideas mediante reglas específicas para justificar un pensamiento.

🛠️ Facultad Resolutiva

La capacidad humana (y ahora artificial) orientada específicamente a la resolución de problemas técnicos.

📑 Resultado Lógico

El conjunto de proposiciones enlazadas que sirven de base para defender o demostrar una tesis.

4.1. Tipos de Razonamiento

Los sistemas inteligentes modernos aplican diferentes arquitecturas según la naturaleza del problema:

Lógica
Basado en Reglas

Habilidad de manejar información para adquirir conocimiento y tomar decisiones deductivas.

Estadística
Probabilístico

Ideal para entornos con incertidumbre (clínicos, financieros) mediante Redes Bayesianas.

Ambigüedad
Razonamiento Borroso

Supera el binario "V/F". Modela la realidad en grados de pertenencia continuos.

Experiencia
Basado en Casos

Soluciona problemas nuevos adaptando soluciones utilizadas en escenarios anteriores.

Evidencia
Razonamiento Evidencial

Métodos como Dempster-Shafer para combinar evidencias de múltiples fuentes externas.

Bio-inspirado
Redes Neuronales

Modelos computacionales que evolucionan a partir de la estructura del cerebro biológico.

4.1.1. Razonamiento Basado en Reglas

🛠️ Concepto

Consiste en establecer predicados que enuncien resultados si se cumplen determinas condiciones previas.

🚀 Aplicación

Es vital en aplicaciones contextuales. Ejemplo: Uso del Generic Reasoner de Apache Jena para inferencia semántica.

# Seudocódigo de una regla lógica en Jena/N3
[regla_alarma: 
    (?sensor :tipo :temperatura) 
    (?sensor :valor ?v) 
    greaterThan(?v, 40) 
    -> (?sensor :estado :alerta)
]

4.1.2. Redes Bayesianas

Son grafos acíclicos dirigidos (DAG) donde los nodos representan variables aleatorias y los arcos simbolizan relaciones causales.

📊 Estructura Matemática

Se representan mediante una Matriz de Adyacencia (1 hay conexión, 0 ausencia).

010
001
000
🎲 Probabilidades Clave
  • A Priori: Nodos raíz sin predecesores.
  • CPT (Tablas Condicionadas): Probabilidades de nodos hijos dado sus padres.

Clasificadores Naïve Bayes

Una particularización eficiente donde los atributos son independientes entre sí, simplificando el grafo a una estructura de árbol.

Ejemplo: Diagnóstico Médico. Las causas (enfermedades) son nodos raíz y los efectos (síntomas) son nodos hijos. Dada la presencia de Y1 e Y2, el sistema calcula la probabilidad de la causa X.

💰 Sector Financiero

Evaluar ingresos y depósitos para inferir el riesgo de impago. (Ingresos altos + Depósito alto = 5% riesgo).

🏥 Sector Salud

Sistemas de triaje online que manejan la incertidumbre de los síntomas para sugerir diagnósticos probables.

4.1.3. Teoría Dempster-Shafer

A diferencia de la probabilidad pura, el razonamiento evidencial trata con conjuntos de hipótesis y no requiere un modelo probabilístico completo.

🛡️ Funciones de Creencia

Utiliza Belief Functions para combinar múltiples fuentes de información (evidencias) y calcular la plausibilidad de un evento.

📡 Fusión de Sensores

Ideal para sistemas IoT donde se fusionan datos de distintos sensores para reducir el número de hipótesis de trabajo.

Deducción Contextual: A diferencia de las redes bayesianas, este método es superior para tratar la incertidumbre en computación contextual gracias a su gestión avanzada de restricciones.

4.1.4. Razonamiento Borroso

La lógica difusa emula el cerebro humano procesando datos subjetivos o imprecisos, alejándose del binario rígido (V/F) de Aristóteles.

🧪 Fuzzificación

Conversión de valores reales en grados de pertenencia a conjuntos difusos.

⚙️ Motor de Inferencia

Aplicación de reglas difusas para procesar relaciones entre variables.

📉 Defuzzificación

Transformación del resultado difuso en un valor concreto de salida.

Ejemplo de Middleware: Detectores de red y Fuzzy Context Composers que monitorizan la calidad de la señal para determinar el estado de la conexión en niveles abstractos.

4.1.5. Redes Neuronales

Sistemas de procesado paralelo que emulan el comportamiento biológico para lograr abstracción y robustez en el razonamiento.

🧠 Arquitectura
  • Monocapa: Una única capa de procesamiento.
  • Multicapa: Conocido como Deep Learning.
🏋️ Entrenamiento

Ajuste de pesos mediante el algoritmo de Retropropagación (Backpropagation) basado en ejemplos etiquetados.

# Función de decisión del Perceptrón
# z = x1*w1 + x2*w2 + ... + xn*wn
output = 1 if sum(inputs * weights) >= threshold else 0

Sabías que... En entornos Smart Home, los datos de sensores pueden entrenar redes neuronales cuyos resultados se aplican luego a motores de razonamiento CBR (Basado en Casos).

4.1.6. Razonamiento Basado en Casos

El CBR (Case-Based Reasoning) resuelve problemas nuevos mediante la analogía con escenarios pasados, adaptando soluciones previas a necesidades actuales.

🔄 Ciclo de Vida

Recuperar casos similares, Adaptar la solución, Revisar el resultado y Retener el nuevo saber.

⚖️ Crítica Técnica

Sus detractores apuntan que acepta la evidencia anecdótica sin un modelado estadístico riguroso para la generalización.

Aplicación Domótica: Ideal para hogares inteligentes donde no se conocen a priori todas las interdependencias entre electrodomésticos y servicios.

Fusión Inteligente: OntoCBR

OntoCBR es una arquitectura híbrida que utiliza el contexto descrito en Ontologías como las características base para los casos del sistema CBR.

🔗 Integración

Combina la riqueza semántica de las ontologías (vocabulario formal) con la capacidad de resolución práctica por analogía del CBR.

💡 Valor Real

Permite abordar problemas reales de forma granular, facilitando la deducción en dominios dinámicos y altamente interdependientes.

4.2. Razonamiento Basado en Casos

El CBR se fundamenta en el modo de actuación humano: utilizar recuerdos y experiencias pasadas para resolver problemas futuros mediante analogía.

✅ Éxito Pasado

Si una solución fue efectiva en un problema similar previo, parece lógico aplicarla (o adaptarla levemente) al escenario actual.

❌ Fracaso Pasado

Si el resultado previo no fue bueno, la opción lógica es modificar la solución para evitar repetir el mismo error.

Clave del Modelo: Problemas similares tienen soluciones similares. En ausencia de base de conocimiento, se recurre a procesos creativos, pero siempre se genera una nueva experiencia aprendida.

El Caso como Lección Aprendida

Un caso es la base del sistema: es una experiencia concreta almacenada que representa una lección tras resolver un problema.

⭐ Juicio de Valor

Es vital evaluar si la solución fue buena. Un experto puede refinar la base de conocimiento para mejorar exponencialmente las capacidades del sistema.

🗺️ De CBR a CBP

Cuando las soluciones propuestas son planes de acción, hablamos de CBP (Case-Based Planning), una especialización del razonamiento por casos.

Nota: El CBP se profundizará en el último punto de esta unidad como técnica avanzada de planificación inteligente.

Ciclo de vida: Las 4 "Rs"

La arquitectura de un sistema CBR se define por un proceso secuencial de aprendizaje continuo:

🔍 1. Retrieve (Recuperar)

Análisis de casos almacenados buscando similitudes con el problema actual (especialmente en la descripción).

♻️ 2. Reuse (Reutilizar)

Adaptar soluciones previas mediante modificación, combinación o selección de la opción más adecuada.

🛠️ 3. Revise (Revisar)

Evaluación de la solución propuesta. ¿Es apropiada? Intervención de expertos para corregir posibles errores.

💾 4. Retain (Retener)

Almacenamiento del nuevo caso resuelto en la Memoria de Casos para generalizar el aprendizaje.

Memoria vs. Conocimiento

📦 Memoria de Casos

Encargada del mantenimiento y organización de las experiencias. Es la fuente de soluciones anteriores.

📚 Base de Saber

Contiene teorías, principios y reglas que asisten en las etapas de Adaptación y Revisión.

Case Based Reasoning: cbrlib

En Python, la librería cbrlib permite integrar el razonamiento basado en casos sobre objetos nativos (dataclasses), calculando similitudes ponderadas.

import dataclasses
from cbrlib import (WeightedPropertyEvaluatorMapping, ReasoningRequest, casebase, evaluate)

@dataclasses.dataclass(slots=True, frozen=True)
class DataObject:
    color: str = None
    size: int = None

# 1. Definir Mapeos de Evaluación
def dataobject_evaluator():
    mappings = (
        WeightedPropertyEvaluatorMapping("color", evaluate.equality, 2),
        WeightedPropertyEvaluatorMapping("size", evaluate.numeric, 1),
    )
    return functools.partial(evaluate.case_average, mappings)

# 2. Inferencia sobre Memoria de Casos
memory = [DataObject(color="red", size=20), DataObject(color="blue", size=50)]
query = DataObject(color="red")
result = casebase.infer(memory, ReasoningRequest(query=query), dataobject_evaluator())

4.3. Sistemas de Planificación

La planificación es un tipo de razonamiento orientado a metas: IA que razona sobre sus propias acciones para alcanzar un estado objetivo.

🎯 Eficaces

El plan propuesto debe resolver el problema con éxito total.

📋 Completos

Se deben valorar todos los planes posibles del espacio.

⚖️ Óptimos

Proponer el plan ideal según criterios de tiempo y recursos.

El Plan: Es la secuencia de acciones que permite transitar desde un estado inicial hasta el estado final objetivo.

4.3.1. Método de Ruta Crítica (CPM)

El Critical Path Method (CPM) determina la duración mínima de un proyecto analizando la secuencia de actividades relacionadas.

📍 Ruta Crítica

Es la trayectoria más larga del proyecto. Cualquier retraso en estas actividades críticas pospone la fecha final.

⏳ Holguras

Margen de tiempo en actividades no críticas que permite retrasos sin afectar al plan global.

Certidumbre: En el CPM, las duraciones son conocidas (sin incertidumbre), lo que facilita el control exhaustivo de costes y avance.

Fases del Método CPM

❶ Planificación

Definición de tareas, establecimiento de precedencias y dibujo de la red de actividades.

❷ Control

Aprobación, monitorización de avances y aplicación de medidas correctivas.

Representación de la Red

Un plan de tareas se visualiza mediante una red de actividades donde cada flecha es una tarea y cada círculo un suceso de control.

🔘 Nodos y Sucesos

Representan instantes de tiempo (inicio/fin). Un suceso precede al inicio de las actividades que salen de él.

👻 Actividades Ficticias

Flechas discontinuas que no consumen tiempo. Sirven para salvar precedencias e individualizar actividades.

Regla de Precedencia (i < j)

Es buena práctica numerar los nodos de forma que el identificador del inicio (i) sea siempre menor que el del fin (j). Esto simplifica el cálculo aritmético posterior.

Tipos de Relaciones

⏸️ Paralelas

Tareas que pueden ejecutarse simultáneamente sin afectación mutua parcial o total.

🔗 Relacionadas

El inicio de una tarea depende estrictamente de la finalización de la precedente.

🔱 Divergentes

Múltiples actividades que nacen de un único evento común inicial.

🔑 Preguntas de Oro

Para construir la red, debemos responder: ¿Qué va antes? ¿Qué va después? ¿Qué puede ir al mismo tiempo?

Tiempos Tempranos (ES)

Cálculo del momento más próximo en el que puede arrancar una tarea (Earliest Start).

ESj = max{ ESi + tij }
Procedimiento Forward
  1. Asignar 0 al suceso inicial.
  2. Sumar duraciones hacia adelante.
  3. En nodos de convergencia, elegir el valor máximo.

Tiempos Tardíos (LS)

Cálculo del momento límite para finalizar una tarea sin retrasar el proyecto (Latest Start).

LSi = min{ LSj - tij }
Procedimiento Backward
  1. Partir de la duración total calculada en ES.
  2. Restar duraciones hacia atrás.
  3. En nodos de divergencia, elegir el valor mínimo.

Holgura y Ruta Crítica

La holgura TF (Time Free) indica el retraso tolerable sin afectar a la fecha final de entrega.

TFij = LSj - ESi - tij
🔴 Ruta Crítica (TF = 0)

Camino donde todas las actividades tienen holgura cero. Es la trayectoria de mínima duración posible.

🟢 Actividades No Críticas

Aquellas con holgura mayor a cero; permiten flexibilidad en el uso de recursos.

Grafo de Actividades (Ejemplo)

Visualización de un plan de tareas con nodos de control e interdependencias críticas.

Ruta Crítica

Secuencia roja: Actividades sin holgura.

Ruta Secundaria

Act. Ficticia

Sincroniza eventos sin sumar tiempo.

🧠 Lógica de Cálculo

Para iniciar la Tarea N, se requiere completar L y M. El tiempo temprano (ES) de N será de 10 días (el máximo de las rutas de entrada). La Tarea M tiene una holgura de 6 días.

4.3.2. Planificación Basada en Casos (CBP)

En el CBP (Case-Based Planning), la solución generada es un plan de acción. Es una tarea de memoria donde planificar equivale a recordar.

🎯 Anticipación

Permite identificar y descartar planes que presentaron problemas en el pasado antes de ejecutarlos.

🛠️ Depuración

Facilita la corrección de planes fallidos basándose en el aprendizaje de errores previos.

💾 Almacenamiento

Los planes exitosos se guardan en una Memoria de Planes para su reutilización inmediata.

Diferencia Clave: A diferencia de otras técnicas que buscan el refinamiento del plan actual, el CBP adapta casos antiguos para resolver problemas nuevos con flexibilidad jerárquica o abstracta.
