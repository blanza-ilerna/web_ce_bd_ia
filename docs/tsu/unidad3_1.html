Representación del Conocimiento (KR)

La base del comportamiento inteligente: formalizar lo que sabemos para que la IA pueda razonar.

KR (Knowledge Representation): Es el área fundamental que estudia cómo el conocimiento simbólico permite a los sistemas IA actuar de forma racional basándose en lo que "saben" sobre su entorno.

Aplicaciones del Razonamiento KR

🩺 Diagnóstico

Detección de causas ante anomalías del sistema basándose en evidencias detectadas.

📅 Planificación

Construcción de secuencias lógicas de acciones para transformar el estado actual en el objetivo deseado.

🤯 Razonamiento Avanzado

Gestión de incertidumbre, previsión de oponentes y actualización dinámica del saber sin contradicciones.

Silogismo Deductivo (Inferencia):
• Premisa Mayor: Los leoneses son españoles.
• Premisa Menor: Los españoles son europeos.
• Conclusión: Los leoneses son europeos (Deducción lógica).

3.2. Representaciones Formales: Lógica

La lógica es el lenguaje declarativo por excelencia en IA. Permite validar afirmaciones y generar nuevo conocimiento mediante sentencias que modelan situaciones reales.

📜 Lenguaje Sintáctico

Define constantes, propiedades y relaciones para establecer conexiones entre conceptos con una semántica precisa.

🧠 Lógica Clásica

Se divide principalmente en Lógica Proposicional (frases simples) y Lógica de Predicados (profundización en sujetos).

3.2.1. Lógica Proposicional

Es el lenguaje lógico más simple. Permite relacionar frases declarativas atómicas (P, Q, R) mediante conectivas lógicas.

^
Y (Conjunción)

Significado: Une dos proposiciones y solo es verdadera si ambas lo son simultáneamente. Equivale al "y" del lenguaje natural.

Ej: "Llueve ^ hace frío" (Solo es verdad si ocurren las dos cosas).

v
O (Disyunción)

Significado: Representa una opción entre dos alternativas. Es verdadera si al menos una de las proposiciones es verdadera.

Ej: "Voy al cine v voy al parque" (Es verdad si voy a cualquiera de ellos).

¬
NO (Negación)

Significado: Invierte el valor de verdad de una proposición. Si algo es verdadero, su negación es falsa.

Ej: "¬P" (Se lee como: "No es cierto que P").

→
Si... entonces (Condicional)

Significado: Establece una relación de causa-efecto. Indica que si se cumple la primera, la segunda debe cumplirse obligatoriamente.

Ej: "Estudio → apruebo" (Si es verdad el antecedente, debe ser verdad el consecuente).

A B A ^ B A v B A → B
VVVVV
VFFVF
FVFVV
FFFFV

Lógica Proposicional (II): Validación

Para garantizar que un razonamiento es correcto, la lógica utiliza metodologías que permiten validar si una conclusión se deriva legítimamente de sus premisas.

🧠 Validación Semántica

Se centra en el significado. Intenta demostrar que para que una conclusión sea válida, debemos partir necesariamente de premisas verdaderas.

📐 Validación Sintáctica

Se basa en la estructura algebraica (Álgebra de Boole). Valida la corrección del enunciado operando con los símbolos independientemente de su significado.

El Álgebra de Boole

Es la base matemática de la lógica computacional. George Boole diseñó un sistema algebraico donde las variables solo pueden tener dos estados: 1 (Verdadero) o 0 (Falso).

En el álgebra de Boole, los conectores lógicos se comportan como operaciones matemáticas:

  • AND (^): Similar a la multiplicación (1 * 1 = 1; cualquier otro caso es 0).
  • OR (v): Similar a la suma lógica (1 + 0 = 1; solo 0 + 0 es 0).
  • NOT (¬): Inversión del valor (0 → 1; 1 → 0).

Este sistema permite simplificar expresiones lógicas complejas mediante reglas estructurales, facilitando que los procesadores ejecuten razonamientos lógicos a gran velocidad.

Práctica: Simbolización de Enunciados

¿Cómo traducimos el lenguaje natural al lenguaje de la lógica proposicional? Analicemos la siguiente proposición:

"O está jugando y corriendo o está saltando en la silla"
📍 1. Atomatización
  • p: está jugando
  • q: está corriendo
  • r: está saltando en la silla
🏗️ 2. Estructuración

Identificamos los conectores:

(p ^ q): jugando Y corriendo (bloque 1)

v r: O saltando (alternativa)

Solución: ((p ^ q) v r)

3.2.2. Lógica de Predicados

La lógica proposicional es limitada para expresar relaciones complejas. La Lógica de Predicados amplía el vocabulario permitiendo añadir parámetros (objetos) a los átomos.

📦 Términos

Variables (x, y...): Sujetos a diferentes valores.

Constantes (a, b...): Valores concretos del universo.

Funciones (f, g...): Devuelven valores basados en parámetros.

🔢 Cuantificadores

Universal (∀x): Se cumple para TODO x en el dominio.

Existencial (∃x): Se cumple para AL MENOS UN x en el dominio.

Lógica de Predicados (II)

Para validar razonamientos en predicados, utilizamos potentes técnicas de álgebra lógica:

🧬 Forma Normal de Skolem

Proceso fundamental en la resolución automática de teoremas. Su objetivo es eliminar los cuantificadores existenciales (∃x) para simplificar la fórmula.

Cálculo: Se sustituye la variable existencial por una constante única (si no está bajo el efecto de un ∀) o por una función de Skolem (si depende de un ∀).

Ejemplo: ∀x ∃y (Padre(y, x)) → Se convierte en: ∀x (Padre(f(x), x))

🤝 Unificación

Sustitución de variables para que dos expresiones diferentes se vuelvan idénticas, permitiendo aplicar reglas de inferencia.

Caso Práctico: Atletas y Pasiones

Analicemos la estructura lógica de este razonamiento:

"Ningún feo despierta pasiones; todos los atletas despiertan pasiones. Por lo tanto, ningún atleta es feo".

📍 1. Definición de Predicados

F(x): x es feo

D(x): x despierta pasiones

A(x): x es atleta

👨‍🏫 2. Formalización Lógica
1. ∀x (F(x) → ¬D(x))
2. ∀x (A(x) → D(x))
C: ∀x (A(x) → ¬F(x))

Límites de la Lógica Clásica

Aunque potente, la lógica de predicados pura tiene dificultades para modelar ciertos aspectos de la realidad humana y empresarial:

🕒 El Tiempo

Dificultad para representar estados pasados o futuros. Requiere Lógicas Temporales específicas.

🤝 Creencias de Otros

La lógica define lo que el sistema sabe, pero no lo que cree que otros saben. Área de las Lógicas Epistémicas.

🎲 Incertidumbre

La lógica clásica es binaria (V/F). La realidad a menudo es imprecisa, requiriendo Lógicas Probabilísticas o Difusas.

3.3. Representaciones Estructuradas

Programar exclusivamente con lógica es como usar lenguaje ensamblador: potente pero complejo de escalar. Las representaciones estructuradas surgen como "lenguajes de alto nivel" para modelar el conocimiento.

Ventaja: Facilitan la adquisición y el mantenimiento de grandes volúmenes de información mediante estructuras organizadas y jerárquicas.

3.3.1. Redes Semánticas

Formalismo basado en un Grafo donde los conceptos son nodos y las relaciones son arcos. Su origen está en la psicología cognitiva: los humanos asociamos conceptos por relaciones.

📊 Diagramas Entidad-Relación

Un ejemplo clásico de red semántica técnica que describe entidades y sus vínculos en una base de datos.

🐍 Implementación (NetworkX)

En Python, la librería NetworkX es el estándar para construir y analizar redes complejas de forma programática, permitiendo deducir relaciones implícitas mediante algoritmos de grafos.

# Ejemplo funcional de Red Semántica en NetworkX
import networkx as nx

# 1. Instanciamos el grafo de conocimiento
G = nx.Graph()

# 2. Definimos relaciones con atributos semánticos
G.add_edge("León", "España", relación="está en")
G.add_edge("España", "Europa", relación="está en")
G.add_edge("Catedral", "León", relación="ubicada en")

# 3. Consultamos la ruta lógica entre conceptos
camino = nx.shortest_path(G, "Catedral", "Europa")
print(f"Ruta de conocimiento: {camino}")
# Salida: ['Catedral', 'León', 'España', 'Europa']

3.3.2. Sistemas de Marcos (Frames)

Los Marcos organizan formalmente la información de las redes semánticas. Son colecciones de atributos que describen detalladamente el comportamiento de los elementos.

🏰 Estructura Taxonómica

Utilizan relaciones especiales como clase/subclase e instancia, permitiendo la herencia de atributos y la jerarquización del saber.

⚙️ Razonamiento Eficiente

Su potencia radica en algoritmos rápidos para consultas de propiedades, aunque a costa de no permitir negaciones o conceptos existenciales complejos.

# Analogía con Programación Orientada a Objetos
class Vehiculo:
    def __init__(self):
        self.ruedas = 4  # Atributo heredado

class Coche(Vehiculo):  # Relación de Herencia (Subclase)
    def arrancar(self):
        print("El frame 'Coche' ejecuta su método")

Anatomía de un Marco

🔠 Atributos

Describen conceptos mediante características y modificadores específicos.

🏃 Métodos

Procedimientos o funciones que realizan cálculos a partir de las instancias accesibles.

🔗 Relaciones

Enlaces bidireccionales (padre-hijo) que organizan los marcos en una estructura coherente.

El problema de la Herencia Múltiple: Si un concepto hereda de varias superclases por distintas rutas, pueden surgir conflictos de atributos que el sistema debe resolver.

3.4. Ontologías

La ontología es el estudio de las entidades existentes y su categorización organizada. Es, en esencia, un vocabulario formal para representar la realidad de un universo específico.

Analogía: Un diccionario es una ontología lingüística; recopila términos, significados y reglas de uso para que los hablantes se comprendan sin ambigüedad.

¿Por qué son vitales en IA?

🚫 Adiós a la Ambigüedad

Permiten que dos agentes inteligentes compartan la misma estructura de información y se comprendan perfectamente.

♻️ Reutilización

Un saber bien definido puede aplicarse a múltiples sistemas sin necesidad de redefinirlo.

📐 Supuestos Explícitos

Obliga a formalizar cada detalle del dominio, haciendo que sea más sencillo analizar y modificar el modelo.

# Ejemplo de Ontología de Dominio: "Ecosistema Móvil"
{
  "Clases": ["Marca", "Modelo", "SistemaOperativo"],
  "Propiedades": {
    "fabricado_por": {"dominio": "Modelo", "rango": "Marca"},
    "ejecuta_en": {"dominio": "Modelo", "rango": "SistemaOperativo"}
  },
  "Instancias": [
    {"ID": "iPhone15", "tipo": "Modelo", "ejecuta_en": "iOS"},
    {"ID": "Apple", "tipo": "Marca"}
  ]

3.4.2. Desarrollo de una Ontología

Definir una ontología es un proceso iterativo que formaliza las clases, propiedades e instancias de un dominio específico.

📋 Requisitos de Definición
  • Identificar las clases del universo.
  • Organizar la jerarquía por afinidades.
  • Indicar propiedades y restricciones (tipo, cardinalidad).
  • Asignar valores para crear instancias.

💡 Sabías que...

No hay una única forma de definir un universo; la solución óptima depende estrictamente del problema a abordar y requiere múltiples iteraciones.

Etapa 1: Alcance y Preguntas de Competencia

Antes de modelar, debemos definir el universo y alcance. Una técnica eficaz es plantear "Preguntas de Competencia": interrogantes que la ontología debe ser capaz de responder para ser considerada funcional.

Etapas de Desarrollo (II)

♻️ 2. Reutilización

Las ontologías establecen vocabularios comunes. Es vital investigar si existen componentes previos para evitar redundancias y asegurar la interoperabilidad.

🧱 3. Elementos y Jerarquía

Listar términos prioritarios y estructurarlos mediante relaciones de generalización y especialización (Taxonomías).

Etapa 4: Propiedades y Características

Definir la estructura interna: cualidades descriptivas, nombres identificadores y relaciones con otras clases. Se deben especificar restricciones como Cardinalidad, Rango e Inversas.

3.4.3. Claves para el Éxito

⚠️ Coherencia Terminológica
  • Evitar mezclar singulares y plurales.
  • Distinguir claramente entre el nombre y la esencia (clase vs. identificador).
  • Mantener un nivel de generalidad uniforme en las subclases.
📐 Estructura Jerárquica
  • Nivel ideal: entre 2 y 12 niveles de profundidad.
  • Evitar ciclos (clase A subclase de B y viceversa).
  • Vigilar la transitividad en las relaciones.

Principio de Granularidad: No incluyas todo el universo. Limita la ontología solo a las clases y atributos básicos necesarios para tu aplicación específica.

Implementación con Owlready2

En el ecosistema Python, Owlready2 es la librería líder para la creación y manipulación de ontologías siguiendo el estándar OWL 2.0. Permite integrar razonadores (HermiT) directamente sobre objetos nativos de Python.

🦉 Ventajas

Convierte clases de la ontología en clases de Python, permitiendo una manipulación intuitiva y el uso de lógica automática para inferir nuevos conocimientos.

🛠️ Estándar

OWL 2.0: Proporciona una semántica rica para definir restricciones complejas y jerarquías profundas de forma interoperable.

from owlready2 import *

# 1. Crear la Ontología en memoria
onto = get_ontology("http://test.org/demo.owl")

with onto:
    # 2. Definir Clases (Marcos)
    class Droga(Thing): pass
    class EfectoSecundario(Thing): pass
    
    # 3. Definir Propiedades
    class tiene_efecto(Droga >> EfectoSecundario): pass

    # 4. Crear Instancia
    aspirina = Droga("Aspirina")
    mareo = EfectoSecundario("Mareo")
    aspirina.tiene_efecto.append(mareo)

# 5. Guardar el resultado en formato OWL (XML)
onto.save(file="farmacia.owl", format="rdfxml")

Resumen de la Unidad

En una sociedad impulsada por los datos, el tratamiento eficaz de la información mediante técnicas de Representación del Conocimiento (KR) es el pilar de los sistemas inteligentes modernos.

Conclusión: El conocimiento se organiza en esquemas que permiten la comprensión, almacenamiento y recuperación de información, actuando como predictores esenciales para el razonamiento automático.

Formalismos Clave Analizados:

⚖️ La Lógica

Lenguaje universal pero complejo. Enfrenta retos con la incertidumbre, el tiempo y el conocimiento de otros agentes.

🏗️ Repr. Estructurada

Cercana al razonamiento humano (redes y marcos). Facilita la adquisición de grandes volúmenes de saber.

🧬 Ontologías

Vocabulario formal para compartir realidades. Vitales para eliminar ambigüedades en sistemas de IA.
