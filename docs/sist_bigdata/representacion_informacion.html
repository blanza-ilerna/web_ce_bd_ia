üìä La Importancia de Visualizar Big Data

¬øPor Qu√© Visualizar?

La visualizaci√≥n de datos transforma conjuntos masivos de informaci√≥n en representaciones gr√°ficas comprensibles que permiten descubrir patrones, detectar anomal√≠as y comunicar insights de forma efectiva. En Big Data, donde trabajamos con millones o miles de millones de registros, la visualizaci√≥n es cr√≠tica para:

  • Toma de decisiones: Un dashboard bien dise√±ado puede revelar tendencias en 3 segundos que tomar√≠an horas analizar en tablas
  • Detecci√≥n de anomal√≠as: Outliers y patrones an√≥malos son visualmente evidentes
  • Comunicaci√≥n ejecutiva: Stakeholders no t√©cnicos comprenden mejor gr√°ficos que SQL queries
  • Exploraci√≥n de datos: Interactive charts permiten drill-down y an√°lisis ad-hoc

üìö Caso de Estudio: Universidad de Oxford - Analytics Dashboard

Contexto: La Universidad de Oxford gestiona datos de 26,000 estudiantes, 7,000 staff y 100+ programas acad√©micos. Su plataforma de analytics procesa 50 GB de datos diarios (registros acad√©micos, asistencia, uso de biblioteca, plataforma Moodle, servicios de carrera).

Desaf√≠o: Los reportes trimestrales tradicionales (400 p√°ginas Excel) tardaban 3 semanas en generarse y nadie los le√≠a completos. La administraci√≥n necesitaba insights en tiempo real para:

  • Identificar estudiantes en riesgo de abandono
  • Optimizar asignaci√≥n de recursos (aulas, profesores, presupuesto)
  • Medir efectividad de programas de apoyo
  • Detectar tendencias de rendimiento por facultad

Soluci√≥n: Dashboard interactivo con Tableau conectado a data warehouse (Snowflake). Incluye:

  • KPI Overview: Tasa de retenci√≥n (94.2%), promedio GPA (3.45), satisfacci√≥n estudiantil (87%)
  • Risk Heatmap: Mapa de calor de estudiantes por riesgo de abandono (algoritmo ML con 89% precisi√≥n)
  • Attendance Trends: Serie temporal de asistencia por programa (detect√≥ ca√≠da 15% en Ingenier√≠a durante ex√°menes)
  • Resource Utilization: Ocupaci√≥n de aulas en tiempo real (optimiz√≥ uso +22%)
  • Graduate Outcomes: Treemap de salarios por carrera (identific√≥ programas con mejor ROI)

Resultados:

  • ‚è±Ô∏è Tiempo de generaci√≥n de reportes: 3 semanas ‚Üí 30 segundos
  • üìà Identificaci√≥n temprana de estudiantes en riesgo: +65% (permiti√≥ intervenciones proactivas)
  • üí∞ Ahorro en recursos: ¬£1.2M/a√±o por optimizaci√≥n de espacios y profesorado
  • üéì Tasa de retenci√≥n mejorada: 92.1% ‚Üí 94.2% (+2.1 puntos porcentuales)
  • üë• Uso del dashboard: 8,500 vistas/mes por staff administrativo y acad√©mico

import plotly.graph_objects as go
import numpy as np
from sklearn.cluster import KMeans
import pandas as pd

# Dataset: MIT - Segmentaci√≥n de 900 estudiantes por patrones de comportamiento
np.random.seed(123)
n_students = 900

# Cluster 1: Estudiantes de alto rendimiento (30%)
n_cluster1 = int(n_students * 0.30)
gpa_c1 = np.random.normal(3.7, 0.2, n_cluster1)
study_c1 = np.random.normal(35, 5, n_cluster1)
engagement_c1 = np.random.normal(85, 8, n_cluster1)

# Cluster 2: Estudiantes promedio equilibrados (50%)
n_cluster2 = int(n_students * 0.50)
gpa_c2 = np.random.normal(3.2, 0.3, n_cluster2)
study_c2 = np.random.normal(22, 6, n_cluster2)
engagement_c2 = np.random.normal(68, 12, n_cluster2)

# Cluster 3: Estudiantes en riesgo (20%)
n_cluster3 = n_students - n_cluster1 - n_cluster2
gpa_c3 = np.random.normal(2.5, 0.4, n_cluster3)
study_c3 = np.random.normal(12, 4, n_cluster3)
engagement_c3 = np.random.normal(45, 15, n_cluster3)

# Combinar datos
gpa_all = np.concatenate([gpa_c1, gpa_c2, gpa_c3])
study_all = np.concatenate([study_c1, study_c2, study_c3])
engagement_all = np.concatenate([engagement_c1, engagement_c2, engagement_c3])

# Aplicar K-Means clustering
X = np.column_stack([gpa_all, study_all, engagement_all])
kmeans = KMeans(n_clusters=3, random_state=42, n_init=10)
clusters = kmeans.fit_predict(X)

# Crear DataFrame
df_students = pd.DataFrame({
    'GPA': gpa_all,
    'Horas_Estudio_Semanal': study_all,
    'Engagement_%': engagement_all,
    'Cluster': clusters
})

# Mapear clusters a etiquetas descriptivas
cluster_labels = {
    0: 'Alto Rendimiento',
    1: 'Promedio Equilibrado',
    2: 'En Riesgo'
}
df_students['Categoria'] = df_students['Cluster'].map(cluster_labels)

# Crear scatter 3D interactivo

colors = ['#2ECC71', '#3498DB', '#E74C3C']  # Verde, Azul, Rojo

    df_cluster = df_students[df_students['Categoria'] == cluster_name]

        x=df_cluster['GPA'],
        y=df_cluster['Horas_Estudio_Semanal'],

        name=cluster_name,
        marker=dict(
            size=6,

        ),

Estudio: {study:.1f}h
Engagement: {eng:.1f}%"

                                          df_cluster['Horas_Estudio_Semanal'],
