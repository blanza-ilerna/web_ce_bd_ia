Error Tipo I — Falso Positivo (FP)

Falsa alarma. El modelo predice positivo cuando es negativo.

Ej: filtro spam bloquea correo legítimo. Detector oncológico da positivo en paciente sano.

Métrica que lo controla: Precision = TP/(TP+FP)

SAAPIA
Error Tipo II — Falso Negativo (FN)

Omisión. El modelo predice negativo cuando es positivo.

Ej: detector de fraude deja pasar transacción fraudulenta. Test médico no detecta enfermedad.

Métrica que lo controla: Recall = TP/(TP+FN)

SAAPIA
Ejemplo Resuelto — Clasificador de Enfermedades (200 pacientes)

TP=60, FP=20, FN=20, TN=100. Total=200

Pred. EnfermoPred. Sano
Real Enfermo60 (TP)20 (FN)
Real Sano20 (FP)100 (TN)
Accuracy = (60+100)/200 = 0.80
Precision = 60/(60+20) = 0.75
Recall = 60/(60+20) = 0.75
Specificity = 100/(100+20) = 0.833
F1 Score = 2×(0.75×0.75)/(0.75+0.75) = 0.75
Accuracy (Exactitud)

% total de predicciones correctas. Falla con clases desbalanceadas (ej: 95% clase A → 95% accuracy prediciendo siempre A).

Acc = (TP + TN) / (TP + TN + FP + FN)
SAAPIASBDBDA
Precision (PPV)

De lo predicho positivo, cuánto es realmente positivo. Penaliza Error Tipo I.
Ej: "De 80 correos marcados spam, ¿cuántos lo eran?"

Precision = TP / (TP + FP)
SAAPIABDA
Recall (Sensibilidad / TPR)

De lo positivo real, cuánto detectó el modelo. Penaliza Error Tipo II.
Ej: "De 80 pacientes enfermos, ¿cuántos detectó?"

Recall = TP / (TP + FN)
SAAPIASBDBDA
Specificity (Especificidad / TNR)

De lo negativo real, cuánto descartó correctamente el modelo.
Ej: "De 120 sanos, ¿a cuántos confirmó como sanos?"

Specificity = TN / (TN + FP)
SAAPIA
F1-Score

Media armónica de Precision y Recall. Ideal con datasets desbalanceados.
Ej: Prec=0.9, Rec=0.1 → F1=0.18 (detecta el desequilibrio que Accuracy esconde).

F1 = 2 × (Prec × Rec) / (Prec + Rec)
SAAPIASBDBDA
AUC-ROC

Área bajo la curva ROC (TPR vs FPR a distintos umbrales). Capacidad global de discriminación.
0.5=azar | 0.7-0.8=aceptable | 0.8-0.9=bueno | >0.9=excelente.

SAAPIA

B. REGRESIÓN

MAE (Mean Absolute Error)

Error absoluto medio. Mismas unidades que el dato. Fácil de interpretar. Robusto a outliers.

MAE = Σ|y_real - y_pred| / n
SAASBDBDA
MSE (Mean Squared Error)

Penaliza severamente los outliers al elevar al cuadrado. También se usa como función de pérdida en DL.

MSE = Σ(y_real - y_pred)² / n
MIASAASBDBDA
RMSE (Root Mean Squared Error)

Raíz del MSE. Devuelve el error a la escala original manteniendo la penalización cuadrática a los outliers.

RMSE = √MSE
SAASBDBDA
R² (Coeficiente de Determinación)

% de varianza que explica el modelo. Rango [0,1]. 1=ajuste perfecto.
Ej: R²=0.85 → el modelo explica el 85% de la varianza.

R² = 1 - (SS_res / SS_tot)
SAABDA

C. NO SUPERVISADO & REGLAS DE ASOCIACIÓN

Silhouette Score

Mide cohesión (cercanía al propio clúster) vs separación (distancia al vecino). Rango [-1, 1]. Mayor es mejor.

SAABDA
Inercia (WCSS)

Suma de distancias al cuadrado al centro del clúster. Se minimiza. Base del método del codo en K-Means.

SAA
Soporte (Support)

Frecuencia de aparición de un itemset en el dataset.

Supp(A) = |A| / |Total|
SAA
Confianza (Confidence)

Probabilidad de B dado que ha ocurrido A.

Conf(A→B) = Supp(A∪B)/Supp(A)
SAA
Lift

Fuerza de la regla sobre el azar. >1=correlación positiva. =1=independencia.

Lift = Conf(A→B) / Supp(B)
SAA

D. DEEP LEARNING — FUNCIONES DE PÉRDIDA

Loss (Pérdida)

Valor de la función de error que se minimiza por backpropagation + gradiente descendente en cada epoch de entrenamiento.

MIASAA
Cross-Entropy (Entropía Cruzada)

Función de pérdida estándar para clasificación neuronal. Compara distribuciones de probabilidad real vs predicha.
— Binaria: 2 clases (sigmoid). — Categórica: N clases (softmax).

MIASAA
IoU (Intersection over Union)

[Visión Artificial] Solapamiento entre Bounding Box real y predicha en detección de objetos (YOLO, SSD).

IoU = Área(intersección) / Área(unión)
MIA
mAP / BLEU / Perplexity

mAP [CV]: Precision media para múltiples clases e IoU.
BLEU [NLP]: Calidad de traducciones automáticas vs referencias.
Perplexity [NLP]: Predicción de texto. Menor = mejor.

MIA

E. RENDIMIENTO BIG DATA

Latencia

Tiempo de respuesta del sistema (ms) al procesar una petición o evento.

SBDBDA
Throughput (Rendimiento)

Eventos, bytes o registros procesados por unidad de tiempo (msg/s, MB/s).

SBDBDA
IOPS

Operaciones de lectura/escritura de disco por segundo. Crítico en HDFS, RAID y almacenamiento distribuido.

SBDBDA

F. CALIDAD DE DATOS & NEGOCIO

Dimensiones de Calidad de Datos
  • Completitud: % de valores no nulos.
  • Unicidad: % de registros no duplicados.
  • Consistencia: Coherencia cruzada entre sistemas.
  • Vigencia (Timeliness): Datos actualizados.
BDA
ROI / Churn Rate / Conversion Rate

ROI: Beneficio del proyecto IA vs su coste total.
Churn Rate: Tasa de abandono de clientes.
Conversion Rate: % usuarios que completan un objetivo.
