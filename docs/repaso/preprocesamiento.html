A. PIPELINE DE PREPROCESAMIENTO — VISIÓN GENERAL

¿Por qué preprocesar?

Los modelos de ML aprenden de los datos: si los datos tienen errores, el modelo los aprende. "Garbage in, garbage out." El preprocesamiento transforma datos crudos en datos limpios, consistentes y en la escala adecuada para que el modelo pueda aprender correctamente.

Datos Crudos → Limpieza → Transformación → Escalado → Feature Eng. → Modelo

Etapa Qué hace Técnicas En el temario
1. Inspección Entender el dataset. Identificar problemas. df.info(), df.describe(), df.isnull().sum(), histogramas. SAA PIA
2. Limpieza Tratar valores nulos, duplicados y atípicos. Eliminación, imputación, winsorización, IQR. SAA PIA SBD
3. Codificación Convertir variables categóricas a numéricas. Label Encoding, One-Hot, Ordinal, Target Encoding. SAA PIA
4. Escalado Homogeneizar escalas de variables numéricas. MinMaxScaler, StandardScaler, RobustScaler. SAA PIA
5. Feature Engineering Crear o seleccionar las mejores variables. PCA, correlación, importancia, creación manual. SAA PIA SBD
6. División Separar Train / Validation / Test. train_test_split, K-Fold, StratifiedKFold. SAA PIA

B. VALORES NULOS Y DATOS FALTANTES

Tipos de datos faltantes
Tipo Siglas Descripción Estrategia
Missing Completely At Random MCAR Ausencia completamente aleatoria. No relacionada con ninguna variable. Imputación simple o eliminación segura.
Missing At Random MAR La ausencia depende de otras variables observables (no del propio valor). Imputación múltiple, modelos predictivos.
Missing Not At Random MNAR La ausencia depende del propio valor no observado. Ej: pacientes graves no responden encuesta. Modelado específico. No eliminar sin análisis.
SAAPIASBD
Estrategia Cuándo Código Python Precaución
Eliminar filas Poco % de nulos (<5%) y MCAR. df.dropna() Puede reducir el dataset significativamente.
Eliminar columnas Columna con >50% nulos. df.dropna(axis=1, thresh=0.5) Perder información potencialmente útil.
Imputar con Media Variables numéricas continuas sin outliers. df.fillna(df.mean()) Sensible a outliers. Distorsiona la distribución.
Imputar con Mediana Variables numéricas con outliers. df.fillna(df.median()) La más recomendada para datos numéricos asimétricos.
Imputar con Moda Variables categóricas. df.fillna(df.mode()[0]) Puede sobrerepresentar la categoría más frecuente.
Imputar con constante Cuando el nulo tiene significado (ej: 0 = sin compras). df.fillna(0) / df.fillna('Unknown') Puede introducir sesgo si el nulo no es 0.
KNN Imputer Pocos nulos, relación con otras variables. KNNImputer(n_neighbors=5) Costoso computacionalmente con datasets grandes.
Imputación predictiva MAR. Relación compleja con otras variables. Entrenar modelo con filas completas para predecir nulos. Riesgo de data leakage si no se hace con cuidado.

C. VALORES ATÍPICOS (OUTLIERS)

Detección de Outliers
Método Cuándo Fórmula / Código
IQR (Intercuartílico) Distribución no normal. General. Outlier si x < Q1−1.5·IQR o x > Q3+1.5·IQR
Z-Score Distribución aproximadamente normal. Outlier si |z| > 3   (z = (x−μ)/σ)
Isolation Forest Multivariante, sin distribución conocida. IsolationForest(contamination=0.05)
Boxplot / Visualización Exploración inicial siempre. df.boxplot() / sns.boxplot()
SAAPIASBD
Tratamiento de Outliers
Estrategia Descripción Cuándo
Eliminar Borrar la fila con el outlier. Error de medición claro, pocos afectados.
Winsorización / Clipping Sustituir por el límite (percentil 1/99). Valor de negocio válido pero extremo.
Imputar Reemplazar por mediana/media. Outlier univariado de origen cuestionable.
Transformación log log(x) comprime la cola larga. Variables con distribución muy sesgada a derecha.
Robustizar el modelo Usar algoritmos no sensibles (árboles). Outliers reales con significado de negocio.

Regla: Entender primero el por qué del outlier antes de actuar. Un valor extremo puede ser información valiosa.

D. ESCALADO DE VARIABLES NUMÉRICAS

¿Por qué escalar?

Algoritmos basados en distancias (KNN, SVM, K-Means) o en gradiente (redes neuronales, regresión logística) son muy sensibles a la escala. Una variable en miles de euros domina sobre una en centímetros. El escalado iguala la influencia de todas las variables.

No necesitan escalado: Árboles de decisión, Random Forest, XGBoost (son invariantes a la escala).

SAAPIA
Técnica Fórmula Rango Cuándo usarla Sensible a outliers Código sklearn
Min-Max Scaling
(Normalización)		 X' = (X−Xmin)/(Xmax−Xmin) [0, 1] Redes neuronales, Imágenes. Distribución no normal. Rango acotado necesario. Sí — un outlier comprime todos los valores. MinMaxScaler()
Standard Scaling
(Estandarización)		 X' = (X−μ)/σ (−∞, +∞) media=0 σ=1 Regresión, SVM, PCA, algoritmos que asumen normalidad. Más habitual en ML clásico. Moderado — mejor que Min-Max pero no robusto. StandardScaler()
Robust Scaling X' = (X−Q2)/(Q3−Q1) Sin límite Dataset con muchos outliers que no puedes eliminar. Usa la mediana (Q2) y el IQR en vez de media/std. No — diseñado para ser robusto a extremos. RobustScaler()
Log Transform X' = log(X+1) (−∞, +∞) Variables muy sesgadas a la derecha (precios, salarios, ingresos). Aproxima a normal. No aplicable — es transformación, no escalado. np.log1p(X)
Power Transform
(Box-Cox / Yeo-Johnson)		 Transformación paramétrica que maximiza normalidad. Variable Cuando se necesita distribución normal y log no es suficiente. Solo valores positivos (Box-Cox). Bajo PowerTransformer()
Regla de Oro: Fit solo en Training, Transform en todo
scaler.fit(X_train) → scaler.transform(X_train) → scaler.transform(X_test)

Nunca hacer fit_transform(X_completo) antes de dividir: introduce data leakage. El escalador debe aprender únicamente los parámetros del Training Set y aplicarlos al Test tal cual.

E. CODIFICACIÓN DE VARIABLES CATEGÓRICAS

Técnica Tipo de variable Cuándo Código Python Problema En el temario
Label Encoding Ordinal (con orden real) Categorías con jerarquía válida: Básico<Medio<Avanzado. Árboles de decisión. LabelEncoder() Introduce orden artificial en nominales. No usar con variables nominales. SAA PIA
One-Hot Encoding Nominal (sin orden) Pocas categorías (<10–15). Estándar para modelos lineales y redes neuronales. pd.get_dummies(df, drop_first=True) Explosión dimensional con muchas categorías. Trampa de variable dummy (usar drop_first=True). SAA PIA
Ordinal Encoding Ordinal Categorías con orden definido. Similar a Label pero con mapeo manual explícito. OrdinalEncoder(categories=[['S','M','L','XL']]) Requiere saber el orden de antemano. SAA PIA
Target Encoding Nominal alta cardinalidad Reemplaza categoría por la media del target. Muy eficaz para variables con muchas categorías. TargetEncoder() (sklearn 1.3+) Riesgo de data leakage si se aplica antes de dividir. Riesgo de overfitting. SAA PIA
Binary Encoding Nominal 10–100 categorías Codifica categoría como número → bits. 100 categorías = solo 7 columnas binarias. ce.BinaryEncoder() Menos interpretable. Depende de librería category_encoders. SAA
Frequency Encoding Nominal alta cardinalidad Reemplaza categoría por su frecuencia en el dataset. Captura la rareza de categorías. df['col'].map(df['col'].value_counts(normalize=True)) Puede perder información si dos categorías tienen la misma frecuencia. SAA
Guía Rápida — ¿Qué Encoding usar?
Situación Técnica recomendada
Variable con orden natural (talla S/M/L) Ordinal Encoding con mapeo explícito.
Nominal con <10 categorías + modelo lineal/red neuronal One-Hot Encoding con drop_first=True.
Nominal con <10 categorías + árbol/bosque Label Encoding es suficiente.
Nominal con 10–100 categorías Binary Encoding o Target Encoding.
Nominal con >100 categorías (códigos postales, IDs) Frequency / Hashing Encoding o embeddings.
Alta cardinalidad + target bien definido Target Encoding con validación cruzada para evitar leakage.

Selección de Variables (Feature Selection)
Método Tipo Descripción
Correlación de Pearson Filtro Elimina variables con correlación >0.9 entre sí (multicolinealidad) o <0.05 con el target.
Chi-cuadrado (χ²) Filtro Para variables categóricas vs categóricas. Mide dependencia estadística.
Importancia (Random Forest) Wrapper/Embedded feature_importances_. Rankea según cuánto reduce la impureza en árboles.
RFE (Recursive Feature Elimination) Wrapper Elimina variables de forma recursiva entrenando el modelo. Lento pero preciso.

PCA Reducción dim. Crea nuevas variables no correlacionadas que maximizan la varianza. Pierde interpretabilidad.

G. DATOS DESBALANCEADOS — CLASES MINORITARIAS

¿Qué es el desequilibrio de clases?

Cuando una clase tiene muchas más muestras que otra. Ej: 99% transacciones normales vs 1% fraudes. El modelo aprende a predecir siempre la clase mayoritaria → 99% accuracy pero 0% recall en fraudes. La accuracy engaña.

SAAPIAMIABDA
Técnica Estrategia Descripción Cuándo
Oversampling Aumentar clase minoritaria Duplicar muestras de la clase minoritaria aleatoriamente. Pocas muestras. Riesgo de overfitting si es naive.
SMOTE Síntesis de muestras Genera muestras sintéticas interpolando entre vecinos de la clase minoritaria (no duplicados exactos). El más usado. Mejor que oversampling simple. Solo en Training.
Undersampling Reducir clase mayoritaria Eliminar muestras aleatoriamente de la clase con más datos. Dataset muy grande. Riesgo de perder información importante.
Class Weights Penalizar modelo Asignar mayor peso al error en la clase minoritaria durante el entrenamiento. Fácil de implementar: class_weight='balanced'. Sin modificar datos.
Umbral de clasificación Post-procesado Bajar el umbral de 0.5 a 0.3 para predecir positivo. Aumenta recall a costa de precisión. Cuando los FN son más caros que los FP (diagnóstico médico).
Stratified K-Fold Evaluación Mantiene proporción de clases en cada fold. Evaluación fiable con clases desbalanceadas. Siempre que se use validación cruzada con clases desbalanceadas.

H. SESGO Y VARIANZA — DIAGNÓSTICO DEL MODELO

Alto Sesgo (Underfitting)

El modelo es demasiado simple para capturar los patrones del problema.

Síntoma: Error alto en Train Y en Val.

Soluciones: Modelo más complejo, más features, menos regularización, más epochs (DL).

SAAPIA
Alta Varianza (Overfitting)

El modelo memoriza el Training Set pero no generaliza.

Síntoma: Error bajo en Train, alto en Val.

Soluciones: Más datos, regularización L1/L2, Dropout, Cross-Validation, Data Augmentation, modelo más simple.

SAAPIA
Equilibrio Sesgo-Varianza

El balance óptimo donde el error de generalización es mínimo.

Síntoma: Error similar en Train y Val. Modelo generaliza bien.

Error = Sesgo² + Varianza + Ruido
SAAPIAMIA
