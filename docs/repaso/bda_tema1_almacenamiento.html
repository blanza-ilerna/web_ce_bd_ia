Leyenda: BDA Big Data Aplicado (Módulo 5075)

A. TIPOLOGÍAS DE ALMACENAMIENTO (DAS, NAS Y SAN)

DAS (Direct Attached)

Almacenamiento directamente conectado al servidor (USB, SATA). Muy rápido y barato, pero no se puede compartir por red. Si el servidor cae, los datos quedan aislados.

NAS (Network Attached)

Almacenamiento compartido conectado a la red TCP/IP (como una "carpeta compartida" gigante). Opera a nivel de archivos (NFS, SMB/CIFS). Ideal para ofimática y archivos compartidos.

SAN (Storage Area Net.)

Red independiente de ultra-alta velocidad (Fibra Óptica/iSCSI) dedicada 100% al almacenamiento. Opera a nivel de bloques puros. Es el estándar para Bases de Datos masivas críticas y clústeres.

Configuraciones RAID (Redundancia y Rendimiento) Escalabilidad en Big Data
RAID 0 (Striping): Suma velocidad, 0% redundancia (Si rompe 1 disco pierdes todo).
RAID 1 (Mirroring): Doble coste, es un espejo 1:1. Máxima seguridad.
RAID 5 (Paridad simple): Requiere min 3 discos. Tolerancia a rotura de 1 disco.
RAID 10: Suma velocidad (Striping) de varios espejos (Mirroring). El más caro/seguro. 		Scale-Up (Vertical): Añades más RAM a la misma placa base. Límite físico rápido.
Scale-Out (Horizontal): Añades millones de ordenadores baratos estándar interconectados. Es el pilar del Big Data. (ej: Cassandra).

B. ECOSISTEMA HADOOP Y DATA LAKES AVANZADOS

Apache Hadoop (La Arquitectura Clásica)

  • HDFS: Sistema de archivos. Trocea archivos enormes en Bloques de 128MB y los reparte y clona por miles de nodos (Replicación x3).
  • MapReduce: El "trabajador". El código va hacia donde está el dato (Map), filtra y luego agrupa todas las respuestas parciales en un resultado final (Reduce).
  • YARN: El orquestador/gerente. Asigna la CPU y RAM necesaria a cada nodo.
  • Modelo WORM: Write Once, Read Many. En HDFS es casi imposible "editar/hacer Update" en una celda existente. El dato inmutable garantiza protección absoluta en bancos.

Data Lakes, Spark y Delta Lake

  • Data Lake: "Lago" gigante donde se vierten petabytes de datos en crudo (sin esquema inicial) abaratando el almacenamiento masivo.
  • Apache Spark: La evolución de Hadoop: procesa la información hasta 100 veces más rápido ya que lo ejecuta todo sobre Memoria RAM y no grabando en discos como MapReduce.
  • Delta Lake: Es la capa que añade "ACID" (Integridad relacional transaccional) a un Data Lake en bruto. Permite hacer Updates, Time Travel (volver a datos de ayer) usando ficheros parquet.

Spark Streaming vs Apache Flink (La Guerra del Tiempo Real):

  • Spark Streaming (Micro-Batching): Realmente no es "tiempo real puro". Recoge los eventos como si fuesen mini-fotos/lotes cada X milisegundos (Latencia ~Segundos). Muy fácil de integrar con código viejo SQL.
  • Apache Flink (True Streaming): Motor nativo por eventos. Procesa gota a gota conforme nace el evento en el mundo físico. Es el estándar de Netflix/Uber (Latencia submilisegundo).

D. CLOUD Y MODELOS DE SERVICIO EN ALMACENAMIENTO

La adopción de tecnologías Cloud (AWS S3, Google Cloud Storage, Azure Blob) habilita el desacoplamiento de almacenamiento y computación, ofreciendo facturación nativa por gigabyte subido y facilitando el uso de servicios gestionados sin adquirir hardware millonario.

  • IaaS en Big Data (Discos Brutos): Te alquilan volúmenes vacíos (Amazon EBS). Tú formateas, particionas, instalas HDFS y cruzas los dedos. Máxima personalización y complejidad.
  • PaaS en Big Data (DDBB Gestionadas): Servicios como Google BigQuery o Amazon RDS. El Cloud asume la responsabilidad si un disco físico se quema y auto-escala la capacidad. Tú solo lanzas queries SQL y subes tus Dataframes y modelos.
  • SaaS en Big Data (Analítica Llave en mano): Como Power BI Cloud o Salesforce Einstein. Entras usando el navegador; el software y las bases de datos están 100% cerrados para ti, tú solo usas los gráficos finales.
