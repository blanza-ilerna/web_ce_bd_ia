A. APRENDIZAJE SUPERVISADO

Definición

El algoritmo aprende de datos etiquetados (pares entrada–salida conocida). Aprende el patrón para predecirlo en datos nuevos sin etiquetar.

Analogía: "Un profesor que muestra los ejercicios con sus soluciones."

Proceso: Datos etiquetados → Entrenamiento → Modelo → Predicción en nuevos datos
MIASAAPIASBDBDA
Clasificación — Salida: Categoría discreta

El modelo asigna cada instancia a una clase (binaria o multiclase).
Ej: Spam/No spam, Benigno/Maligno, Perro/Gato/Conejo.

Regresión — Salida: Valor numérico continuo

El modelo predice un número real.
Ej: Precio vivienda €285.000, Temperatura 24.5°C, Duración viaje 42 min.

Algoritmos de Clasificación

AlgoritmoIdea claveCuándo usarloEn el temario
Regresión Logística Función sigmoide σ(z)=1/(1+e⁻ᶻ) → probabilidad 0-1. A pesar del nombre, clasifica. Datos linealmente separables. Diagnóstico, churn, crédito. SAA PIA MIA BDA
Árbol de Decisión Serie de preguntas jerárquicas (if/else). Muy interpretable. Riesgo de overfitting. Aprobación préstamos, diagnóstico médico, segmentación. MIA SAA PIA SBD BDA
KNN (K-Nearest Neighbors) Clasifica según mayoría de los K vecinos más cercanos. Sin entrenamiento explícito ("perezoso"). Recomendaciones, reconocimiento de patrones. K impar evita empates. SAA BDA
SVM (Support Vector Machine) Busca el hiperplano que maximiza el margen entre clases. Kernels para no-lineal. Alta dimensionalidad (texto, bioinformática). Lento con datasets grandes. MIA SAA PIA SBD BDA
Naive Bayes Teorema de Bayes asumiendo independencia entre features. P(C|X)=P(X|C)·P(C)/P(X). Filtros de spam, NLP, clasificación de texto. Muy rápido, pocos datos. MIA SAA PIA SBD
Random Forest Ensemble de N árboles → votación. Reduce overfitting (bagging). Mide importancia de features. Fraude, clasificación de clientes, enfermedades. Alta precisión general. MIA SAA PIA SBD BDA
Gradient Boosting (XGBoost, LightGBM…) Árboles secuenciales (boosting) que corrigen errores del anterior. Máxima precisión, lento. Competiciones de ML, datos tabulares complejos, predicción de riesgo. MIA SAA PIA SBD

Algoritmos de Regresión

AlgoritmoFórmula / Idea claveCuándo usarloEn el temario
Regresión Lineal Simple / Múltiple y = β₀ + β₁x₁ + … + βₙxₙ
Minimiza la suma de errores cuadráticos (OLS). Base de todo el resto.		 Precios inmobiliarios, pronóstico de ventas, salarios, tendencias. SAA PIA SBD BDA
Regresión Polinómica y = β₀ + β₁x + β₂x² + β₃x³ + …
Añade potencias de X para capturar curvas. Cuidado con grado alto → overfitting.		 Trayectorias físicas, crecimiento poblacional, curvas no lineales suaves. SAA
Ridge — Regularización L2 Coste = MSE + λΣβ²
Penaliza el cuadrado de los coeficientes. Los reduce pero NO los elimina. Bueno cuando todas las features son relevantes.		 Multicolinealidad. Muchas features correlacionadas. Cuando no quieres descartar ninguna. SAA PIA BDA
Lasso — Regularización L1 Coste = MSE + λΣ|β|
Penaliza el valor absoluto. Puede llevar coeficientes a exactamente 0 → selección automática de features.		 Cuando sospechas que pocas features son realmente relevantes (sparse). Feature selection implícita. SAA PIA BDA
Elastic Net — L1 + L2 Coste = MSE + λ₁Σ|β| + λ₂Σβ²
Combina Ridge y Lasso. Parámetro r controla el balance entre ambas penalizaciones. Más estable que Lasso solo.		 Datasets con muchas features y correlación entre ellas. Mejor generalización que Lasso o Ridge por separado. SAA BDA
SVR (Support Vector Regression) Versión de SVM para regresión. Define un margen de tolerancia ε alrededor de la predicción. Relaciones no lineales complejas. Robusto a outliers dentro del margen ε. SAA PIA
Random Forest Regressor Ensemble de árboles de regresión. Promedia predicciones de N árboles. Reduce varianza. Alta precisión con relaciones complejas. Robusto a outliers. Datos tabulares. SAA PIA SBD
Gradient Boosting Reg. (XGBoost…) Árboles secuenciales que corrigen residuos del anterior. Minimiza función de pérdida personalizable (MSE, MAE…). Máxima precisión en datos tabulares. Competiciones. Predicción de demanda, precios. SAA PIA

B. APRENDIZAJE NO SUPERVISADO

Definición

El algoritmo trabaja con datos sin etiquetas. Descubre por sí mismo la estructura, patrones y relaciones ocultas.

Analogía: "Un radar que descubre lo que no sabías que existía."

Se usa como preprocesamiento antes del supervisado (ej: reducir dimensiones con PCA antes de entrenar un clasificador).

MIASAAPIABDA
1. Clustering (Agrupamiento)

Divide datos en grupos donde los elementos del mismo grupo son más similares entre sí que con los de otros.

2. Reducción de Dimensionalidad

Reduce el número de variables preservando la mayor información posible. Útil para visualización y aceleración de modelos.

3. Detección de Anomalías

Identifica puntos que se desvían del patrón normal ("lo raro"). Crítico en fraude, ciberseguridad y mantenimiento.

Algoritmos de Clustering

AlgoritmoIdea claveVentaja / LimitaciónEn el temario
K-Means Asigna cada punto al centroide más cercano e itera hasta convergencia. Hay que especificar K. Usa el método del codo para elegir K. Rápido y escalable. Solo detecta clusters esféricos. Sensible a outliers. MIA SAA PIA SBD BDA
DBSCAN Agrupa puntos por densidad. Parámetros: ε (radio) y MinPts. Etiqueta outliers como "ruido". No necesita K. Detecta formas arbitrarias y outliers. Sensible a ε y MinPts. MIA SAA PIA SBD BDA
Hierarchical Clustering Crea un árbol (dendrograma) de clusters. Aglomerativo (abajo→arriba) o divisivo. No requiere K fijo. Costoso computacionalmente en grandes datasets. SAA SBD
GMM (Gaussian Mixture) Modelado probabilístico: cada punto pertenece a cada cluster con cierta probabilidad. Clusters blandos (soft assignment). Más flexible que K-Means. SAA

Reducción de Dimensionalidad y Anomalías

AlgoritmoTipoIdea claveEn el temario
PCA Reducción Proyección lineal a componentes principales (máxima varianza). Seleccionar componentes que expliquen 80-95% de varianza. SAA PIA SBD BDA
t-SNE / UMAP Reducción (visual.) Visualización de datos de alta dimensión en 2D/3D. UMAP es más rápido y escalable que t-SNE. SAA PIA SBD BDA
Isolation Forest Anomalías Las anomalías se aíslan más rápido que los puntos normales con árboles aleatorios. SAA SBD BDA
One-Class SVM Anomalías Aprende la frontera de los datos normales. Fuera de ella → anomalía. SAA SBD
LOF (Local Outlier Factor) Anomalías Compara densidad local con vecinos. Detecta outliers locales (no solo globales). SAA SBD BDA

C. APRENDIZAJE POR REFUERZO (Reinforcement Learning)

Definición y Ciclo Fundamental

Un agente aprende a tomar decisiones interactuando con un ambiente. Recibe recompensas (positivas) o penalizaciones (negativas) según sus acciones. Objetivo: maximizar la recompensa acumulada, no solo la inmediata.

Ciclo: Estado (s) → Acción (a) → Recompensa (r) + Nuevo Estado (s') → Actualizar política (π)

Analogía: aprender a montar en bici — sin manual, solo prueba, error y equilibrio ganado.

MIASAAPIA

Conceptos Clave

ConceptoSímboloDefiniciónEjemplo
Agente—El aprendiz / tomador de decisiones.Robot, coche autónomo, jugador de ajedrez.
Ambiente—El mundo donde opera el agente.Tablero, carretera, simulación.
EstadosSituación actual del ambiente percibida por el agente.Posición del robot, configuración del tablero.
AcciónaMovimiento o decisión ejecutable en el estado actual.Mover arriba/abajo, acelerar/frenar.
RecompensarSeñal numérica que evalúa la acción.+10 por ganar, −1 por chocar.
Políticaπ(s)→aEstrategia que mapea estados a acciones."En el estado s, toma la acción a".
Factor descuentoγPeso de recompensas futuras. γ≈0=cortoplacista, γ≈1=largoplacista.γ=0.9 → recompensas futuras valen 90% de las inmediatas.
MDP — Proceso de Decisión de Markov

Formalización matemática del RL. El futuro depende solo del estado presente, no del pasado (Propiedad de Markov).

MDP = (S, A, P, R, γ)  —  S: estados | A: acciones | P: probabilidades de transición | R: función de recompensa | γ: descuento
Ecuación de Bellman: V(s) = max[R(s,a) + γ Σ P(s'|s,a) V(s')]
Explotación vs Exploración

Explotación: usar el conocimiento actual → máxima ganancia a corto plazo.
Exploración: probar acciones nuevas → posible mejora a largo plazo.

Estrategia: ε-Greedy — con prob. ε explora, con (1-ε) explota. Reducir ε a lo largo del tiempo.

Desafíos del RL
  • Alto coste computacional (millones de interacciones).
  • Inestabilidad en el entrenamiento.
  • Diseño de la función de recompensa es un arte.
  • Sim-to-Real Gap: lo que funciona en simulación puede fallar en el mundo real.

Algoritmos Principales de RL

AlgoritmoTipoIdea claveCasos de uso
Q-Learning Off-policy, model-free Aprende la función Q(s,a) que estima el valor de cada acción en cada estado. Guarda valores en una Q-Table.
Q(s,a) ← Q(s,a) + α[r + γ maxQ(s',a') − Q(s,a)]		 Juegos simples, navegación en espacios discretos, optimización de rutas.
DQN (Deep Q-Network) Off-policy + Deep Learning Sustituye la Q-Table por una red neuronal. Añade Experience Replay y Target Network para estabilidad. Juegos Atari, robots con percepción visual, trading.
Policy Gradient (PPO, A2C…) On-policy Aprende directamente la política óptima en lugar de los valores Q. Funciona con espacios de acción continuos. Drones, brazos robóticos, juegos complejos (Dota2, StarCraft II).
Actor-Critic Híbrido Actor = política (decide qué hacer). Critic = valor (evalúa qué tan buena fue la decisión). Combina ventajas de Policy Gradient y Value Function. Control continuo, asistentes conversacionales.
