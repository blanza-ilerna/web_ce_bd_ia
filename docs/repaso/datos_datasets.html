B. DATASETS — CONJUNTOS DE DATOS

¿Qué es un Dataset?

Colección organizada de datos usada para entrenar, validar y evaluar modelos. Cada fila = instancia/muestra. Cada columna = feature/variable. La columna objetivo = target (y).

MIASAAPIASBDBDA
Calidad del Dataset
  • Completitud: sin valores nulos críticos.
  • Consistencia: sin duplicados ni contradicciones.
  • Representatividad: cubre todos los casos reales.
  • Balance: clases no desproporcionadas.
  • Actualidad: datos recientes y vigentes.

C. CONJUNTOS DE ENTRENAMIENTO — TRAIN / VAL / TEST

¿Por qué dividir el dataset?

Evaluar y entrenar con los mismos datos genera evaluación sesgada: el modelo memoriza (overfitting) y parece funcionar bien pero falla con datos nuevos. La solución: dividir en 3 subconjuntos independientes.

Dataset completo → [ Training Set ] [ Validation Set ] [ Test Set ]
SAAPIAMIASBD
Conjunto% TípicoPropósitoRegla claveEn el temario
Training Set 60 – 80% El modelo aprende: ajusta parámetros internos (pesos, coeficientes, reglas). El más grande. Riesgo de overfitting si el modelo es muy complejo. SAA PIA MIA
Validation Set 10 – 20% Ajustar hiperparámetros y comparar modelos durante el desarrollo. Influye indirectamente en el modelo (sesgo indirecto). Se puede usar múltiples veces. SAA PIA
Test Set 10 – 20% Evaluación final imparcial del modelo. Simula datos reales de producción. ¡Solo una vez! Usar varias veces contamina la evaluación (data leakage). SAA PIA MIA

K-Fold Cross-Validation — Validación Cruzada

Cuando el dataset es pequeño, un validation set fijo "desperdicia" datos. K-Fold los aprovecha todos:

  1. Divide el training set en K particiones (folds). Típico: K=5 o K=10.
  2. Entrena K veces: en cada vuelta, una partición diferente actúa como validación.
  3. Promedia las K métricas → estimación más robusta del rendimiento.
  4. El test set queda completamente separado y se usa solo al final.
Accuracy_final = (acc_fold1 + acc_fold2 + acc_fold3 + acc_fold4 + acc_fold5) / K

Variantes: Stratified K-Fold (mantiene proporción de clases) · Leave-One-Out (LOO, K=N, costoso) · Time Series Split (para datos temporales, sin mezclar pasado/futuro).

D. FORMAS DE ENTRENAMIENTO

FormaDescripciónCuándo usarloEn el temario
Entrenamiento por Lotes
(Batch Training)		 El modelo ve todos los datos de training en cada época antes de actualizar los pesos. Estable pero lento y necesita mucha memoria. Datasets que caben en memoria. Cuando la estabilidad de gradiente es crítica. SAA PIA MIA
Mini-Batch Training Se divide el training en lotes pequeños (típico: 32, 64, 128 muestras). Actualiza pesos después de cada mini-lote. El estándar en Deep Learning. Redes neuronales. Balance ideal entre velocidad y estabilidad. GPU friendly. SAA PIA MIA
SGD — Stochastic Gradient Descent Actualiza pesos después de cada muestra individual. Muy ruidoso pero puede escapar de mínimos locales. Datasets gigantes. Online learning. Con learning rate muy pequeño. SAA PIA
Online Learning El modelo actualiza sus pesos con cada nuevo dato que llega (streaming). Aprende de forma continua en producción. Datos en streaming. Sistemas con distribución que cambia en el tiempo (concept drift). SAA SBD BDA
Transfer Learning Se usa un modelo preentrenado en gran dataset (ImageNet, Common Crawl) y se ajusta (fine-tunning) para una tarea específica con pocos datos. Pocos datos etiquetados. NLP (BERT, GPT). Visión (ResNet, VGG). Ahorra tiempo y recursos. MIA SAA PIA
Federated Learning El modelo se entrena en dispositivos locales sin compartir los datos brutos. Solo se comparten los gradientes/pesos. Preserva privacidad. Datos sensibles (salud, banca). Dispositivos móviles (Gboard). RGPD compliance. MIA SBD
Aprendizaje Activo
(Active Learning)		 El modelo selecciona las muestras más informativas para ser etiquetadas por un humano. Minimiza el coste de etiquetado. Etiquetado manual muy costoso. Pocos datos iniciales. Medicina, legal, calidad industrial. MIA SAA
Data Augmentation Genera nuevas muestras sintéticas a partir de las existentes aplicando transformaciones. Aumenta artificialmente el tamaño del dataset. Pocas imágenes o audios. Rotaciones, flips, ruido, recortes. SMOTE para datos tabulares desbalanceados. MIA SAA PIA

TérminoDefinición

Underfitting vs Overfitting
ProblemaSíntomaSolución
Underfitting
(Sub-ajuste)		 Error alto en train Y en val. Modelo demasiado simple. Modelo más complejo, más features, más epochs, menos regularización.
Overfitting
(Sobre-ajuste)		 Error bajo en train, alto en val. Modelo memoriza. Regularización (L1/L2), Dropout, Early Stopping, más datos, Data Augmentation.
Bien ajustado Error similar en train y val. Generaliza bien. — Objetivo del entrenamiento.
