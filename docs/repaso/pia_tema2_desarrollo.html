Cheatsheet: Tema 2. Desarrollo de Aplicaciones de IA

Leyenda: PIA Programación de Inteligencia Artificial (Módulo 5073)

A. ENTORNOS DE MODELADO DE IA (PIPELINE DE DATOS)

1. Recolección de Datos

Las IAs no son nada sin datos. Se usa Scrapy para web scraping, APIs REST conectadas a la nube o bases de datos directas para hacer Data Ingestion de datos estructurados o no estructurados.

2. Manipulación (Pandas)

Proceso para curar el Dataset: imputación de valores nulos, estandarización númerica, eliminación de clases sucias o unificación a través de pipelines.

3. Modelado Evaluativo

Se expone el modelo a herramientas como Scikit-learn, haciendo un particionado (Train/Test) para que pueda aprender de ellos, optimizar hiperparámetros y evaluar métricas como el MSE o Accuracy final.

B. REDES NEURONALES: TENSORFLOW Y MÓDULOS PREDEFINIDOS

Familia de Red Neuronal Comportamiento, Arquitectura ideal y Tipo de Entradas habitual
MLP (Multilayer Perceptron) La red densa más tradicional Feed-Forward. Estupenda para resolver problemas clásicos si los datos son tabulares y estructurados (ej: predecir precio de los pisos).
CNN (Convolucionales) Excelentes a la hora de procesar imágenes. Analizan trozos reducidos haciendo "barridos" jerárquicos. Ideales para clafisicación de objetos y píxeles en sistemas de Visión.
RNN / LSTM (Recurrentes) Redes dotadas de un bucle "memoria". Se especializan en contextos sobre líneas de tiempo, como audios y secuencias en la bolsa bursátil.
Transformers Arquitectura paralela que usa Self-Attention. Destaca por relacionar cada elemento con todos los demás de fondo y en paralelo. Es el estándar por excelencia actual en NLP (texto masivo).

Módulos de Entrenamiento y Transfer Learning

  • Módulos predefinidos: En vez de codificar un modelo desde cero, cargamos modelos pre-entrenados por grandes empresas (ej: ResNet, YOLO o BERT) mediante TensorFlow o Keras. Ahorran meses de cálculo.
  • Feature Extraction: Cargamos el modelo pre-entrenado, "congelamos" TODAS sus capas previas e insertamos una capa final (cabecera) propia para que las predicciones cuadren con nuestro dominio a bajo coste.
  • Fine-Tuning: Descongelamos varias (o todas) las capas superiores del modelo original para readaptar minuciosamente sus pesos con nuestros datos. Gasta más coste pero más preciso.

Componentes Auxiliares Keras PIA

  • Optimizador (AdamW / SGD): Decide "cómo y a qué ritmo" alteraremos los pesos y corregiremos el rumbo.
  • Función de Pérdida (Loss Factor): Juez matemático que calcula empíricamente cuán equivocada es la predicción de la IA en cada Epoch.
  • Schedulers: Decelera el Learning Rate del modelo de forma progresiva automáticamente.
  • Callbacks (Early Stopping): Automatizaciones capaces de interrumpir el entrenamiento de golpe si el modelo entra en sobreajuste (overfitting).

C. GENERACIÓN DE CÓDIGO E IA GENERATIVA (GenAI)

La revolución del desarrollo de software no pasa por ignorar el código, sino aprovechar modelos masivos (LLMs) para automatizar la sintaxis por nosotros. Se debe auditar siempre posibles alucinaciones (código inventado o erróneo).

Herramienta / Categoría ¿Qué es y cómo se usa profesionalmente para programar?
VibeCoding / Agentes Autónomos La evolución total. Entornos como Cursor, Windsurf o Lovable entienden todo tu proyecto y editan/crean ficheros simultáneamente leyendo "tu intención general". Programan por ti en bloque de forma iterativa.
Asistentes IA (Co-Pilots) Extensiones (ej: GitHub Copilot, Tabnine) usadas frecuentemente dentro del IDE clásico (VS Code). Efectúan autocompletado avanzado línea a línea ahorrando horas de búsquedas simples.
Plataformas Low-Code / No-Code Herramientas de interfaces visuales enfocadas a un usuario más de 'Negocio'. La IA interpreta lenguaje natural y te autoconfigura menús y bloques, limitando sin embargo la personalización dura en la arquitectura.

D. FRAMEWORKS, APIs REST Y GESTIÓN DE COSTES APLICADA

Orquestación (LangChain y LlamaIndex)

  • APIs GenAI (OpenAI): Todo interactúa mediante endpoints REST. La API envía el texto ("Prompt") y devuelve la respuesta. Es inútil sin integración de un buen framework en backend.
  • LangChain: El rey de orquestación en Python. Permite la creación de cadenas cognitivas que dotan al LLM de uso de herramientas (navegar, usar API externa) y memoria en bucle.
  • LlamaIndex: Extremadamente sofisticado para gestionar inyecciones documentales (Sistemas RAG) de cara a que el LLM extraiga información sobre tu propio PDF o base de datos empresarial.
Facturación, Tokens y Costes
  • Tokens: La IA generativa no facturiza "por palabra" sino por fragmentos silábicos (aprox. 1 Token ≈ 0.75 a 4 caracteres ingleses). Todo el prompt (entrada y salida) gasta saldo.
  • Optimización (Pruning): Fundamental reducir Data Leakage o dar mucho contexto que no sirve de nada, para evitar reventar el presupuesto Cloud (gastos imprevistos de API Limit).
  • Seguridad (API Keys): Una clave OPENAI_API_KEY jamás debe ser hardcodeada en el repositorio GIT de cara al código público. Obligatorio el uso de variables de entorno .env.
