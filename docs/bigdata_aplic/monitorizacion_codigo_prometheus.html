Configuración de Prometheus y Grafana

Implementación técnica para la recolección de métricas del clúster

Laboratorio de Monitorización

A continuación, se presenta un ejemplo de configuración para monitorear un clúster Hadoop con Prometheus y visualizar métricas en Grafana. Esta configuración permite definir los intervalos de recolección y los puntos finales (endpoints) de los diferentes servicios del ecosistema.

Archivo de Configuración: prometheus.yml

Este archivo es el corazón de la monitorización. Define qué servicios vigilar, con qué frecuencia recolectar los datos y dónde enviar las alertas en caso de anomalías:

/etc/prometheus/prometheus.yml 
global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: 'hadoop_datanodes'
    static_configs:
      - targets: ['datanode1:9100', 'datanode2:9100', 'datanode3:9100']
        labels:
          group: 'hdfs'

  - job_name: 'spark_executors'
    static_configs:
      - targets: ['spark-driver:9100']
        labels:
          group: 'spark'

  - job_name: 'yarn_resource_manager'
    static_configs:
      - targets: ['resourcemanager:9100']
        labels:
          group: 'yarn'

alerting:
  alertmanagers:
    - static_configs:
      - targets: ['alertmanager:9093']

Desglose de la Configuración

Para entender cómo Prometheus interactúa con el clúster Big Data, debemos analizar sus tres bloques principales:

🌍 Configuración Global (Global)

Define los parámetros por defecto para toda la instancia:

  • scrape_interval: Frecuencia con la que Prometheus "tira" de los datos desde los exporters. 15s es un estándar equilibrado para Big Data.
  • evaluation_interval: Cada cuánto tiempo se comprueban las reglas de alerta definidas sobre las métricas recolectadas.

🎯 Configuración de Recolección (Scrape Configs)

Aquí listamos los "objetivos" que queremos monitorizar organizados por tipos de trabajo (jobs):

  • Targets: Listado de IPs y puertos donde residen los Node Exporters (puerto 9100) o exporters específicos del servicio.
  • Labels: Etiquetas personalizadas (como group: 'hdfs') que permiten filtrar y agrupar métricas fácilmente en los paneles de Grafana.
  • Static Configs: Método manual para definir nodos, ideal para clústeres estables o entornos de aprendizaje.

🔔 Sistema de Alertas (Alerting)

Indica a Prometheus dónde debe enviar las notificaciones cuando se dispara una alerta:

  • Alertmanager: Servicio independiente que recibe las alertas, las gestiona (evitando duplicados) y las encamina a Slack o correo.
  • Puerto 9093: Puerto por defecto para la comunicación con el gestor de alertas.

Recomendaciones de Despliegue

Frecuencia de Muestreo

Configurado en 15 segundos para obtener una resolución alta de las métricas sin saturar el ancho de banda dedicado a los datos.

Observabilidad Holística

Al agrupar HDFS, Spark y YARN, Grafana puede mostrar la causa raíz de un fallo correlacionando capas físicas y lógicas.

💡 En un entorno productivo, se recomienda usar Service Discovery (como Kubernetes SD o Consul) en lugar de static_configs para escalar automáticamente cuando se añadan nuevos nodos.
