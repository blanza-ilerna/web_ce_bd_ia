Validación de Calidad con Great Expectations

Asegurando la fiabilidad de los datos en entornos productivos

¿Qué es Great Expectations (GX)?

Great Expectations es la herramienta estándar de código abierto en Python para validar, documentar y perfilar datos. Su filosofía principal es tratar la calidad de los datos como código ("Data Quality as Code"), lo que permite automatizar pruebas y generar documentación que se mantiene siempre actualizada.

🔍 Validación Automatizada

Evita que datos corruptos ("silent failures") rompan tus pipelines o contaminen tus reportes de BI. Si los datos no cumplen las reglas, el pipeline se detiene.

📄 Data Docs

Genera documentación HTML legible para humanos directamente desde tus pruebas. Traduce código complejo en un reporte visual de "Qué datos tenemos" y "Qué esperamos de ellos".

Casos de Uso Principales

🛡️ Protección de Pipelines ETL

Validar datos justo después de la ingesta (Bronze layer) antes de transformarlos. Si llegan nulos excesivos o formatos erróneos, se alerta al equipo.

🤖 Machine Learning

Asegurar que las "features" de entrenamiento mantengan su distribución estadística. Evita el "Training-Serving Skew" validando los datos de inferencia.

🔄 Migración de Datos

Doble validación: asegurar que el número de filas y valores agregados en el sistema destino coinciden exactamente con el sistema origen.

Expectativas Comunes (Functionalities)

La librería ofrece más de 50 expectativas predefinidas ("Expectations"). Aquí tienes las más esenciales para empezar:

expect_table_row_count_to_equal

📏 Volumen: Verifica que el número total de filas sea exactamente el esperado (e.g. validación post-migración).

expect_column_values_to_be_unique

🔑 Unicidad: Detecta duplicados no deseados. Fundamental para claves primarias (PKs) y IDs de transacciones.

expect_column_values_to_not_be_null

🚫 Completitud: Asegura que no existan valores nulos en campos críticos. Permite configurar un umbral de tolerancia (mostly).

expect_column_values_to_match_regex

📝 Formato: Valida que los strings cumplan patrones complejos como emails, DNIs, teléfonos o códigos postales.

expect_column_mean_to_be_between

📊 Distribución: Validación estadística avanzada. Alerta si la media (o desviación) de una columna numérica se desvía de lo esperado.

⌨️ Comandos CLI Importantes

Aunque se puede usar como librería, su CLI (Command Line Interface) es muy potente para configurar proyectos:

# Inicializar un proyecto de Data Quality
great_expectations init

# Conectar a una fuente de datos (CSV, SQL, Spark, Pandas)
great_expectations datasource new

# Crear una nueva Suite de Expectativas (Profiling automático)
great_expectations suite new

# Crear checkpoints para validar datos
great_expectations checkpoint new

# Construir y abrir la documentación HTML
great_expectations docs build

💻 Implementación Práctica en Python

A continuación, implementaremos un script de validación que carga un dataset de ventas, define reglas de negocio críticas y genera un reporte de resultados. 

import great_expectations as ge
import pandas as pd
from great_expectations.dataset import PandasDataset

# 1. Cargar dataset desde HDFS o archivo local
# Simulamos carga de datos crudos
data = pd.read_csv("hdfs://namenode:8021/data/ventas.csv")

# Convertimos el DataFrame de Pandas a un Dataset de Great Expectations
df = ge.from_pandas(data)

# 2. Definir expectativas de calidad

# Completitud: Permitimos un margen de error del 5% de nulos (mostly=0.95)
df.expect_column_values_to_not_be_null(column="precio", mostly=0.95)

# Validez: El precio debe ser un número positivo sensato
df.expect_column_values_to_be_between(column="precio", min_value=0)

# Unicidad: Cada venta debe tener un ID único (sin duplicados)
df.expect_column_values_to_be_unique(column="id_venta")

# Consistencia: La región debe pertenecer a nuestro mapa operativo
df.expect_column_values_to_be_in_set(
    column="region",
    value_set=["US", "EU", "APAC"]
)

# 3. Ejecutar validación
validation_result = df.validate()

# 4. Generar informe y tomar decisiones
if validation_result["success"]:
    print("✅ Todas las validaciones pasaron con éxito")
else:
    print("❌ Errores de calidad detectados:")
    print(validation_result)

# 5. Guardar resultados en HDFS (Formato JSON)
with open("hdfs://namenode:8021/quality_reports/ventas_report.json", "w") as f:
    f.write(str(validation_result))

🔍 Análisis del Script

1. Carga y Conversión (ge.from_pandas)

Cargamos los datos normalmente con Pandas, pero el paso clave es envolverlos con ge.from_pandas(). Esto añade al DataFrame los "superpoderes" de validación, permitiéndole entender métodos como expect_....

2. Reglas de Negocio (Expectativas)

  • mostly=0.95: Es una característica "fuzzy". Acepta que los datos no sean perfectos; la prueba pasa si al menos el 95% de las filas tienen precio.
  • in_set: Valida integridad referencial básica (Domain Integrity), asegurando que no aparezcan regiones inventadas como "MARS".

🛑 El Objeto validation_result

La variable validation_result es un diccionario JSON rico en información. No solo dice success: false, sino que detalla:
• Qué expectativa falló.
• Cuántas filas fallaron (element_count, unexpected_count).
• Muestra de los valores erróneos (partial_unexpected_list), lo cual facilita enormemente la depuración.

📚 Resumen

Implementar Great Expectations transforma la calidad de datos de un proceso reactivo ("¡El reporte está roto!") a uno proactivo. Al integrar este script en tu orquestador (como Airflow), garantizas que solo los datos que cumplen tus estándares de calidad fluyan hacia tus sistemas de análisis.
