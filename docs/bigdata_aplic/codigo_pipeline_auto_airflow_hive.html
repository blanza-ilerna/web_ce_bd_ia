Pipeline Automático: Airflow + Hive

Orquestación de procesos ETL y Análisis de Datos

Automatización en Big Data

🌬️ Apache Airflow

Es una plataforma de código abierto para crear, programar y monitorear flujos de trabajo (pipelines) de manera programática. En Airflow, los flujos se definen como código Python, lo que permite un mantenimiento, versionado y pruebas más sencillos. Usa el concepto de DAG (Directed Acyclic Graph) para gestionar dependencias entre tareas.

🐝 Apache Hive

Como hemos visto, Hive es un sistema de data warehouse sobre Hadoop que facilita la lectura, escritura y administración de grandes conjuntos de datos almacenados en almacenamiento distribuido mediante SQL. Es ideal para tareas de análisis batch sobre grandes volúmenes de datos.

🎯 ¿Por qué usarlos juntos?

La combinación es poderosa: Airflow actúa como el director de orquesta, decidiendo cuándo y en qué orden se ejecutan las tareas, mientras que Hive actúa como el motor de procesamiento pesado, ejecutando las consultas SQL complejas sobre el cluster. Esto permite automatizar reportes diarios, procesos ETL complejos y actualizaciones de Data Lakes sin intervención manual.

💻 Código del Pipeline (DAG)

A continuación se muestra un ejemplo completo de un DAG de Airflow que primero importa datos desde una base de datos MySQL usando Sqoop y luego realiza un análisis de ventas utilizando Hive. 

from airflow import DAG
from airflow.operators.bash import BashOperator
from airflow.operators.hive_operator import HiveOperator
from datetime import datetime, timedelta

# Definir argumentos del DAG
default_args = {
    'owner': 'data_team',
    'depends_on_past': False,
    'retries': 1,
    'retry_delay': timedelta(minutes=5),
}

# Definir DAG
with DAG(
    'sales_pipeline',
    default_args=default_args,
    description='Pipeline para análisis de ventas',
    schedule_interval=timedelta(days=1),
    start_date=datetime(2025, 10, 13),
    catchup=False,
) as dag:

    # Tarea 1: Importar datos con Sqoop
    import_data = BashOperator(
        task_id='import_sales',
        bash_command='sqoop import --connect jdbc:mysql://localhost/sales_db --table ventas --target-dir /data/ventas_hdfs --split-by id_venta',
    )

    # Tarea 2: Analizar datos con Hive
    analyze_data = HiveOperator(
        task_id='analyze_sales',
        hql="""
        SELECT region, SUM(precio) AS total_ventas
        FROM ventas
        GROUP BY region
        """,
        hive_cli_conn_id='hive_conn',
    )

    # Definir dependencias
    import_data >> analyze_data

🔍 Explicación Paso a Paso

Analicemos qué hace exactamente cada bloque del código anterior:

1. Importaciones y Configuración

Importamos las librerías necesarias, incluyendo los operadores específicos (BashOperator para comandos de terminal y HiveOperator para consultas Hive). Definimos default_args para establecer reglas globales como el propietario del proceso y la política de reintentos (1 reintento tras 5 minutos si falla).

2. Definición del DAG

Instanciamos el objeto DAG con el nombre 'sales_pipeline'.

  • schedule_interval: Se ejecutará una vez al día (days=1).
  • start_date: Comienza a ejecutarse a partir del 13 de octubre de 2025.
  • catchup=False: Evita que Airflow ejecute tareas pasadas si el DAG se activa después de la fecha de inicio.

3. Tarea 1: Importación (ETL)

La tarea import_data usa BashOperator para ejecutar un comando de sistema. En este caso, llama a Apache Sqoop para traer datos de una tabla MySQL (ventas) y volcarlos en HDFS. Es el paso de "Extracción" y "Carga" inicial.

4. Tarea 2: Análisis (Hive)

La tarea analyze_data usa HiveOperator. Ejecuta una consulta HQL (Hive Query Language) que agrega las ventas por región. Este cálculo se realiza sobre los datos que acabamos de importar. Requiere una conexión configurada en Airflow (hive_conn).

🔗 Definición de Dependencias

La línea final import_data >> analyze_data es crucial. Utiliza el operador de bitwise shift (>>) para establecer que la tarea de análisis NO debe comenzar hasta que la importación de datos haya finalizado con éxito.

📚 Resumen

Este script representa un flujo de trabajo típico en ingeniería de datos: Ingesta → Procesamiento. Airflow garantiza que si la ingesta falla, el procesamiento no se ejecute (evitando cálculos erróneos o vacíos) y notifique al equipo, proporcionando robustez y confiabilidad al ecosistema de Big Data.
