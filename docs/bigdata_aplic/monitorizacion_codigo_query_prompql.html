Consultas PromQL y Alertas Dinámicas

Extrayendo valor de las métricas mediante el lenguaje de Prometheus

El Lenguaje de las Métricas

PromQL (Prometheus Query Language) es la herramienta fundamental para analizar las series temporales recolectadas del clúster. No solo permite visualizar datos en tiempo real, sino que es la base para definir alertas inteligentes que identifiquen anomalías antes de que el sistema falle.

Ejemplos de Consultas y Reglas de Alerta

A continuación, se muestran ejemplos de cómo monitorizar la tasa de lectura en HDFS y cómo disparar una alerta crítica ante el uso excesivo de memoria en Spark:

consultas_y_alertas.promql 
# Tasa de bloques HDFS leídos en los últimos 5 minutos
rate(hadoop_datanode_blocks_read_total[5m])

# Alerta si uso de memoria Spark supera 90%
ALERT SparkMemoryHigh
  IF rate(spark_executor_memory_used[5m]) > 0.9
  FOR 10m
  LABELS { severity = "critical" }
  ANNOTATIONS {
    summary = "High memory usage in Spark executor",
    description = "Memory usage exceeds 90% for {{ $labels.instance }}."
  }

Dominando la Sintaxis de PromQL

PromQL es un lenguaje de consulta funcional que permite filtrar y agregar datos de series temporales. Su potencia reside en la capacidad de combinar selectores precisos con operadores potentes.

🔍 Selectores y Emparejadores

Permiten filtrar métricas por sus etiquetas (labels):

  • {job="spark"}: Igualdad exacta.
  • {group!="hdfs"}: Desigualdad.
  • {env=~"prod.*"}: Emparejamiento por expresión regular.
  • {node!~"test.+"}: Exclusión por expresión regular.

📊 Vectores y Tiempo

  • Vectores Instantáneos: El valor más reciente (ej. up{job="hadoop"}).
  • Vectores de Rango: Histórico de valores (ej. node_memory_usage[1h]).
  • Offset: Desplazamiento temporal (ej. metric offset 1d para ver el valor de ayer).

Operadores y Funciones Esenciales

Categoría Ejemplo de Sintaxis Uso en Big Data
Aritmética (mem_used / mem_total) * 100 Calcular porcentajes de uso en NameNodes.
Comparación cpu_usage > 85 Detectar saturación en DataNodes.
Agregación sum by (cluster) (up) Contar nodos activos por cada clúster diferente.
Lógica metric_a and metric_b Correlacionar fallos en Spark y YARN a la vez.

rate() vs irate()

rate() calcula el promedio en un rango (estable). irate() se fija en los dos últimos puntos, ideal para detectar picos instantáneos de red.

predict_linear()

Predice el valor futuro basándose en la tendencia. Ejemplo: predict_linear(disk_free[4h], 3600) < 0 avisa si el disco se llenará en 1 hora.

topk() / bottomk()

Devuelve los N elementos con mayor/menor valor. Ejemplo: topk(5, cpu_usage) muestra los 5 nodos más saturados del clúster.

Configuración del Dashboard en Grafana

Para visualizar estas consultas de forma profesional en Grafana, sigue estos pasos fundamentales:

  1. Añadir Data Source: Configurar Prometheus como fuente primaria (habitualmente en http://prometheus:9090).
  2. Editor de Consultas: Insertar la query PromQL (ej. sum(rate(kafka_messages[5m]))) directamente en el panel.
  3. Configurar Thresholds: Establecer umbrales visuales (colores) para que el gráfico cambie a amarillo o rojo según la criticidad.
  4. Prometheus Alertmanager: Integrar las reglas ALERT definidas en PromQL con el sistema de notificaciones Slack o correo.
