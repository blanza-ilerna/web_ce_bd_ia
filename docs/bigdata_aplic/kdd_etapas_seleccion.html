Selección y Adquisición de Datos 📥

La base del descubrimiento de conocimiento

Identificar y recolectar los activos de información adecuados es el primer paso crítico para garantizar el éxito del proceso KDD.

Fase 1: Identificación y Recolección

En esta etapa inicial, el objetivo es localizar y extraer los datos que realmente aportarán valor al análisis. Dada la naturaleza del Big Data, esto implica trabajar con una enorme variedad de fuentes: Bases de datos relacionales (SQL), sistemas NoSQL, archivos de logs masivos, APIs externas y extensos Data Lakes.

La selección no consiste solo en "copiar" datos, sino en discernir qué variables y periodos temporales son necesarios para resolver el problema de negocio planteado.

Tecnologías y Técnicas de Ingesta

Dependiendo del origen y la velocidad de los datos, empleamos diferentes estrategias técnicas:

Queries SQL Avanzadas

Utilizadas para extraer información estructurada de bases de datos relacionales y almacenes de datos (Data Warehouses), permitiendo filtrar y agregar datos en el origen.

Ingesta en Streaming con Kafka

Ideal para capturar eventos en tiempo real, como el comportamiento de usuarios en una web o señales de sensores IoT, garantizando que el flujo de datos no se interrumpa.

Scraping y Consumo de APIs REST

Mediante scripts en Python, las organizaciones pueden recolectar datos de redes sociales, servicios meteorológicos o mercados financieros conectándose a APIs públicas o privadas.

🏬 Caso Real: Omnicanalidad en Retail

Un gran retailer necesita construir un modelo de recomendación personalizado. Para ello, selecciona y unifica datos de tres mundos distintos:

  • Ventas Históricas: Extraídas de un cluster Hive.
  • Comportamiento Web: Capturado en tiempo real a través de Kafka.
  • Datos Demográficos: Consultados en una base de datos MongoDB (NoSQL).

Esta selección multi-fuente es la que permite que el sistema de recomendación sea realmente efectivo al tener una visión 360º del cliente.

📍 Punto Clave

Si la selección es deficiente, el resto del proceso KDD está condenado al fracaso. Recolectar datos irrelevantes o incompletos solo consume recursos sin generar conocimiento útil para la empresa.
