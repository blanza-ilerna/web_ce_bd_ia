Pipeline de Análisis de Logs con PySpark

Procesamiento distribuido para la observabilidad del clúster

Objetivo del Script

Este ejemplo práctico muestra cómo utilizar PySpark para ingerir miles de registros históricos almacenados en HDFS (en formato JSON), parsear los eventos complejos y extraer métricas críticas sobre bottlenecks y patrones de fallo recurrentes.

log_analyzer.py
from pyspark.sql import SparkSession
from pyspark.sql.functions import col, explode, from_json
import json

spark = SparkSession.builder.appName("LogAnalysis").getOrCreate()

# Cargar logs de YARN/Spark desde HDFS (formato JSON)
logs_df = spark.read.json("hdfs://namenode:8020/logs/yarn-history/*.json")

# Parsear eventos de jobs
events = logs_df.select("events").withColumn("event", explode(col("events")))
parsed_events = events.select("event.time", "event.type", "event.status", "event.error")

# Análisis de bottlenecks: Filtrar jobs con shuffle > 50% tiempo
bottlenecks = parsed_events.filter((col("type") == 'shuffle') & (col("status") == "slow")) \
    .groupBy("time").count().orderBy(col("count").desc())

bottlenecks.show(20)

# Detección de patrones de fallo: Conteo de errores recurrentes
failures = parsed_events.filter(col("status") == "failed") \
    .groupBy("error").count().orderBy(col("count").desc())
failures.show(10)

# Guardar resultados para dashboard
bottlenecks.write.mode("overwrite").parquet("hdfs://namenode:8020/analysis/bottlenecks/")
failures.write.mode("overwrite").parquet("hdfs://namenode:8020/analysis/failures/")

spark.stop()

Desglose del Proceso

1. Ingesta Masiva

Utilizamos spark.read.json con wildcards (*) para leer miles de archivos históricos de forma paralela directamente desde el almacenamiento distribuido HDFS.

2. Aplanado de Datos

La función explode transforma arrays de eventos anidados en filas individuales, permitiendo filtrar cada evento por tipo (shuffle, task, stage) de forma granular.

3. Detección de Fallos

Agrupamos por el campo error para identificar cuáles son los fallos más frecuentes, permitiendo priorizar el mantenimiento del hardware o el tuning de software.

4. Persistencia en Parquet

Guardamos los resultados filtrados en formato Parquet, ideal para ser consumido posteriormente por herramientas de visualización como Grafana o Tableau.

📈 Impacto en la Operación
Al automatizar este script, el tiempo de diagnóstico forense se reduce de horas a minutos. El administrador recibe un informe diario con los 10 nodos más inestables y los tipos de cuellos de botella más comunes en las últimas 24 horas.
