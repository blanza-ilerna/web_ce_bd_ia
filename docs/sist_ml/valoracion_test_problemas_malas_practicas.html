6.2.2 Problemas y Malas Prácticas

Evitando el autoengaño: fugas de datos y errores de concepto.

El gran enemigo: Data Leakage

El principal riesgo asociado al uso incorrecto del conjunto de test es el Data Leakage (fuga de datos). Consiste en la filtración de información del test hacia el proceso de entrenamiento o validación.

Este fenómeno puede ser evidente o sutil, pero siempre tiene el mismo efecto: el modelo parece rendir mucho mejor de lo que realmente lo hará ante datos nuevos. Al romper la separación sagrada entre entrenamiento y evaluación, las métricas dejan de ser fiables.

Errores comunes en el Preprocesamiento

Muchos errores técnicos esconden fallos conceptuales graves sobre la independencia de los datos.

❌ Mala Práctica: Global Scaler

Aplicar transformaciones (normalización, estandarización) sobre todo el dataset antes de dividirlo. Los parámetros de escala contienen información del test.

✅ Buena Práctica: Fit only Train

Ajustar el escalador (fit) solo con el entrenamiento. Luego, aplicar esa misma escala (transform) al test. Así se mantiene el test "ciego".

Ejemplo: Selección de Variables

Si eliges las variables más correlacionadas con el objetivo usando todos los datos antes del split, estás incorporando conocimiento del futuro (del test) en la construcción del modelo. El resultado es un modelo artificialmente preciso.

Sesgos en la Toma de Decisiones

🏆 El Test como Árbitro de Modelos

Evaluar varios modelos sobre el test y elegir el mejor. Esto convierte al test en un "pseudo-conjunto de validación". La selección introduce sesgo porque el test influye indirectamente en la elección final.

🔄 Reutilización de Resultados

Modificar el modelo porque el resultado en test ha sido malo y volver a probar con el mismo test. El modelo acaba adaptándose a las particularidades de ese conjunto concreto, perdiendo su valor referencial.

Atención

El error más importante es no entender que el conjunto de test es exclusivamente para evaluar, no para construir. Si se usa el test para ajustar parámetros o comparar arquitecturas, el procedimiento metodológico es incorrecto.
