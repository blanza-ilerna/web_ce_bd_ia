Codificación de Variables Ordinales

Preservando la jerarquía lógica de los datos cualitativos.

El Objetivo: Mantener el Orden

Cuando trabajamos con variables que poseen una jerarquía natural (como niveles de estudio o grados de satisfacción), el reto de la codificación es transformar las etiquetas en números sin perder esa relación de orden.

La técnica consiste en asignar un valor numérico entero único a cada categoría, de manera que la magnitud del número refleje fielmente su posición en la secuencia.

Caso Práctico: Satisfacción del Cliente

Imagina que analizamos una encuesta donde los clientes califican el servicio como "Malo", "Medio" o "Bueno".

Configuración del Mapeo

Para que el modelo entienda que "Bueno" es superior a "Malo", definimos una escala progresiva:

  • 🔴 Malo → 0
  • 🟡 Medio → 1
  • 🟢 Bueno → 2
encoding_ordinal.py 
# Definimos el diccionario de mapeo
satisfaccion_mapping = {
    'Malo': 0, 
    'Medio': 1, 
    'Bueno': 2
}

# Aplicamos la transformación a la columna del DataFrame
df['satisfaccion_cod'] = df['satisfaccion'].map(satisfaccion_mapping)

Consideraciones Técnicas

⚠️ El problema de la distancia uniforme

Esta técnica asume implícitamente que la "distancia" entre categorías es constante. Sin embargo, en la realidad, el salto cualitativo entre "Malo" y "Medio" puede ser mucho mayor que entre "Medio" y "Bueno".

Si la diferencia real entre niveles no es uniforme, el uso de valores enteros (0, 1, 2) podría introducir una distorsión matemática en modelos sensibles a la magnitud, como las regresiones lineales o las redes neuronales.

🏁 Resumen

La codificación ordinal es sencilla y eficiente para mantener jerarquías, pero requiere que el científico de datos sea consciente de las distancias subjetivas entre categorías. En el siguiente apartado veremos qué ocurre cuando no existe orden: la codificación nominal.
