Reducci√≥n de Dimensionalidad

Algoritmos para Reducci√≥n de Caracter√≠sticas en Machine Learning

¬øPor Qu√© Reducir Dimensionalidad?

La reducci√≥n de dimensionalidad es el proceso de disminuir el n√∫mero de caracter√≠sticas (variables) en un dataset mientras se preserva la informaci√≥n m√°s relevante. En Big Data, donde trabajamos con datasets de miles o millones de caracter√≠sticas, esta t√©cnica es fundamental.

Los algoritmos de reducci√≥n combaten la "maldici√≥n de la dimensionalidad": en espacios de alta dimensi√≥n, los puntos se vuelven equidistantes, los modelos overfittean y el tiempo de c√≥mputo se dispara exponencialmente.

Ejemplo Real:

Google Translate procesa textos con vocabularios de 500,000+ palabras. Usa embeddings (Word2Vec) para reducir cada palabra a 300 dimensiones, permitiendo procesar 100 billones de traducciones/a√±o con precisi√≥n del 94% en 133 idiomas.

Beneficios Clave

‚ö° Velocidad

Reduce tiempo de entrenamiento de horas a minutos. Menos dimensiones = menos c√°lculos.

üìä Visualizaci√≥n

Permite graficar datos de alta dimensi√≥n en 2D/3D para an√°lisis exploratorio visual.

üéØ Precisi√≥n

Elimina ruido y caracter√≠sticas irrelevantes, mejorando la generalizaci√≥n del modelo.

üíæ Almacenamiento

Reduce requisitos de memoria. Dataset de 10,000 ‚Üí 100 dimensiones = 99% menos espacio.

PCA: Principal Component Analysis

üéØ Fundamento Matem√°tico

PCA es el algoritmo de reducci√≥n m√°s utilizado. Encuentra componentes principales: nuevos ejes ortogonales que maximizan la varianza de los datos. Transforma caracter√≠sticas correlacionadas en componentes independientes ordenados por importancia.

Paso 1: Centrar datos

Restar la media de cada caracter√≠stica (media = 0)

Paso 2: Matriz covarianza

Calcular correlaciones entre caracter√≠sticas

Paso 3: Autovectores

Obtener direcciones de m√°xima varianza

Paso 4: Proyecci√≥n

Transformar datos a nuevos ejes PC

Complejidad: O(n¬∑d¬≤)

n=muestras, d=dimensiones

üíª Implementaci√≥n PCA en Python

from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler
from sklearn.datasets import load_digits
import matplotlib.pyplot as plt
import numpy as np

# Cargar dataset de d√≠gitos (64 dimensiones)
digits = load_digits()
X = digits.data  # 1797 muestras, 64 caracter√≠sticas (8x8 pixels)
y = digits.target

print(f"Dimensiones originales: {X.shape}")
# Output: (1797, 64)

# Paso 1: Estandarizar datos (importante para PCA)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Paso 2: Aplicar PCA para reducir a 2 componentes
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)

print(f"Dimensiones reducidas: {X_pca.shape}")
# Output: (1797, 2)

# Analizar varianza explicada
varianza = pca.explained_variance_ratio_
print(f"PC1 explica: {varianza[0]*100:.2f}% de varianza")
print(f"PC2 explica: {varianza[1]*100:.2f}% de varianza")
print(f"Total retenido: {sum(varianza)*100:.2f}%")

# Paso 3: Determinar n√∫mero √≥ptimo de componentes
# Retener 95% de varianza
pca_full = PCA(n_components=0.95)
X_95 = pca_full.fit_transform(X_scaled)
print(f"Componentes para 95% varianza: {X_95.shape[1]}")

# Visualizaci√≥n 2D
plt.figure(figsize=(10, 6))
scatter = plt.scatter(X_pca[:, 0], X_pca[:, 1], c=y,
                     cmap='tab10', alpha=0.6, edgecolors='k')
plt.xlabel(f'PC1 ({varianza[0]*100:.1f}% varianza)')
plt.ylabel(f'PC2 ({varianza[1]*100:.1f}% varianza)')
plt.title('PCA: D√≠gitos 64D ‚Üí 2D')
plt.colorbar(scatter, label='D√≠gito (0-9)')
plt.grid(alpha=0.3)
plt.show()

# Reconstrucci√≥n inversa (opcional)
X_reconstructed = pca.inverse_transform(X_pca)
print(f"Error reconstrucci√≥n MSE: {np.mean((X_scaled - X_reconstructed)**2):.4f}")

‚öñÔ∏è Ventajas y Limitaciones

‚úÖ Ventajas

  • R√°pido y escalable (O(nd¬≤))
  • Elimina correlaciones
  • Reduce ruido
  • Interpretable (varianza explicada)

‚ö†Ô∏è Limitaciones

  • Asume relaciones lineales
  • Sensible a escala (requiere normalizaci√≥n)
  • Pierde interpretabilidad original
  • No supervisado (ignora etiquetas)

t-SNE: Visualizaci√≥n No Lineal

üåê Principio de Funcionamiento

t-SNE (t-Distributed Stochastic Neighbor Embedding) es un algoritmo de reducci√≥n no lineal optimizado para visualizaci√≥n. Preserva la estructura local de los datos, agrupando puntos similares cerca y separando los distintos.

Par√°metro: perplexity

Balance entre estructura local/global. T√≠picamente 5-50 (30 por defecto).

Par√°metro: learning_rate

Velocidad de optimizaci√≥n. Rango: 10-1000 (200 t√≠pico).

Par√°metro: n_iter

Iteraciones de optimizaci√≥n. M√≠nimo 1000, ideal 2500+.

Complejidad: O(n¬≤)

Lento para n > 10,000 (usar PCA primero)

üíª Implementaci√≥n t-SNE

from sklearn.manifold import TSNE
from sklearn.datasets import load_digits
from sklearn.preprocessing import StandardScaler
import matplotlib.pyplot as plt
import time

# Cargar datos
digits = load_digits()
X, y = digits.data, digits.target

# Escalar (importante)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Para datasets grandes (>10k): reducir con PCA primero
# from sklearn.decomposition import PCA
# pca = PCA(n_components=50)
# X_pca = pca.fit_transform(X_scaled)

# Aplicar t-SNE
print("Ejecutando t-SNE (puede tardar minutos)...")
start_time = time.time()

tsne = TSNE(
    n_components=2,        # Casi siempre 2D o 3D
    perplexity=30,         # Ajustar seg√∫n tama√±o dataset
    learning_rate=200,     # Learning rate
    n_iter=2500,           # M√°s iteraciones = mejor calidad
    random_state=42,
    verbose=1              # Ver progreso
)

X_tsne = tsne.fit_transform(X_scaled)

print(f"Tiempo: {time.time() - start_time:.2f}s")
print(f"Divergencia KL final: {tsne.kl_divergence_:.4f}")

# Visualizaci√≥n mejorada
plt.figure(figsize=(12, 10))
scatter = plt.scatter(
    X_tsne[:, 0], X_tsne[:, 1],
    c=y,
    cmap='tab10',
    alpha=0.7,
    s=50,
    edgecolors='black',
    linewidth=0.5
)
plt.colorbar(scatter, label='D√≠gito (0-9)')
plt.title('t-SNE: Visualizaci√≥n de D√≠gitos Manuscritos', fontsize=16)
plt.xlabel('Dimensi√≥n t-SNE 1', fontsize=12)
plt.ylabel('Dimensi√≥n t-SNE 2', fontsize=12)
plt.grid(alpha=0.3)
plt.tight_layout()
plt.show()

# COMPARACI√ìN: PCA vs t-SNE
from sklearn.decomposition import PCA

fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(16, 7))

# PCA
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)
ax1.scatter(X_pca[:, 0], X_pca[:, 1], c=y, cmap='tab10', alpha=0.6)
ax1.set_title(f'PCA: {sum(pca.explained_variance_ratio_)*100:.1f}% varianza')
ax1.set_xlabel('PC1')
ax1.set_ylabel('PC2')

# t-SNE
ax2.scatter(X_tsne[:, 0], X_tsne[:, 1], c=y, cmap='tab10', alpha=0.6)
ax2.set_title('t-SNE: Estructura local preservada')
ax2.set_xlabel('Dimensi√≥n 1')
ax2.set_ylabel('Dimensi√≥n 2')

plt.tight_layout()
plt.show()

Otros M√©todos de Reducci√≥n

üìä LDA: Linear Discriminant Analysis

LDA es un m√©todo supervisado que maximiza la separaci√≥n entre clases. A diferencia de PCA, LDA usa las etiquetas de clase para encontrar proyecciones que discriminen mejor entre categor√≠as.

üéØ Objetivo LDA

Maximiza varianza inter-clase, minimiza varianza intra-clase

üìà Componentes LDA

M√°ximo: min(n_clases-1, n_features). Ej: 10 clases ‚Üí max 9 LDs

‚úÖ Ventaja Principal

Mejora rendimiento de clasificadores al enfocarse en separabilidad

from sklearn.discriminant_analysis import LinearDiscriminantAnalysis
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score

# Dataset Iris (4 dimensiones, 3 clases)
iris = load_iris()
X, y = iris.data, iris.target

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42
)

# LDA: Reducir a 2 componentes (max = 3-1 = 2)
lda = LinearDiscriminantAnalysis(n_components=2)
X_train_lda = lda.fit_transform(X_train, y_train)
X_test_lda = lda.transform(X_test)

print(f"Dimensiones: 4D ‚Üí 2D")
print(f"Varianza explicada por LD: {lda.explained_variance_ratio_}")

# LDA tambi√©n puede clasificar directamente
y_pred = lda.predict(X_test)
print(f"Accuracy: {accuracy_score(y_test, y_pred)*100:.2f}%")

# Visualizaci√≥n
import matplotlib.pyplot as plt
plt.scatter(X_train_lda[:, 0], X_train_lda[:, 1], c=y_train, cmap='viridis')
plt.xlabel('LD1')
plt.ylabel('LD2')
plt.title('LDA: Iris Dataset')
plt.show()

üß† Autoencoders: Deep Learning

Los autoencoders son redes neuronales que aprenden representaciones comprimidas de datos mediante codificaci√≥n-decodificaci√≥n. El cuello de botella (bottleneck) fuerza la compresi√≥n, creando una representaci√≥n latente de baja dimensi√≥n. 

import tensorflow as tf
from tensorflow import keras
from tensorflow.keras import layers
import numpy as np

# Cargar MNIST (28x28 im√°genes = 784 pixeles)
(x_train, _), (x_test, _) = keras.datasets.mnist.load_data()
x_train = x_train.astype('float32') / 255.0
x_test = x_test.astype('float32') / 255.0
x_train = x_train.reshape(-1, 784)
x_test = x_test.reshape(-1, 784)

# Definir arquitectura Autoencoder
encoding_dim = 32  # Reducci√≥n: 784 ‚Üí 32 (24x compresi√≥n)

# Encoder
input_img = keras.Input(shape=(784,))
encoded = layers.Dense(256, activation='relu')(input_img)
encoded = layers.Dense(128, activation='relu')(encoded)
encoded = layers.Dense(encoding_dim, activation='relu')(encoded)  # Bottleneck

# Decoder
decoded = layers.Dense(128, activation='relu')(encoded)
decoded = layers.Dense(256, activation='relu')(decoded)
decoded = layers.Dense(784, activation='sigmoid')(decoded)

# Modelo completo
autoencoder = keras.Model(input_img, decoded)

# Encoder separado (para extraer representaci√≥n latente)
encoder = keras.Model(input_img, encoded)

# Compilar
autoencoder.compile(
    optimizer='adam',
    loss='binary_crossentropy',
    metrics=['mse']
)

# Entrenar (reconstruir entrada)
history = autoencoder.fit(
    x_train, x_train,  # Input = Output (auto-supervisi√≥n)
    epochs=50,
    batch_size=256,
    validation_data=(x_test, x_test),
    verbose=1
)

# Extraer representaciones latentes (32D)
X_encoded = encoder.predict(x_test)
print(f"Reducci√≥n: 784D ‚Üí {X_encoded.shape[1]}D")

# Visualizar reconstrucciones
import matplotlib.pyplot as plt
decoded_imgs = autoencoder.predict(x_test[:10])

fig, axes = plt.subplots(2, 10, figsize=(15, 3))
for i in range(10):
    # Original
    axes[0, i].imshow(x_test[i].reshape(28, 28), cmap='gray')
    axes[0, i].axis('off')
    # Reconstruido
    axes[1, i].imshow(decoded_imgs[i].reshape(28, 28), cmap='gray')
    axes[1, i].axis('off')
plt.suptitle('Original (arriba) vs Reconstruido (abajo)')
plt.show()

üí° Dato Curioso: Word2Vec y la Analog√≠a de Einstein

En 2013, Google desarroll√≥ Word2Vec, un algoritmo que reduce vocabularios de 500,000+ palabras a embeddings de 300 dimensiones. Lo sorprendente: en este espacio comprimido, las relaciones sem√°nticas se preservan como operaciones vectoriales.

üßÆ √Ålgebra de Palabras:

vector("Rey") - vector("Hombre") + vector("Mujer") ‚âà vector("Reina")

vector("Par√≠s") - vector("Francia") + vector("Italia") ‚âà vector("Roma")

vector("Einstein") - vector("Cient√≠fico") + vector("Pintor") ‚âà vector("Picasso")

Esta propiedad emerge autom√°ticamente de la reducci√≥n dimensional sin programaci√≥n expl√≠cita de reglas sem√°nticas.

üåê GPT-4 usa embeddings de 12,288 dimensiones, reduciendo textos complejos a vectores que capturan significado, contexto e incluso sarcasmo. Esta compresi√≥n inteligente es clave para procesar 25,000+ palabras de contexto en tiempo real.
