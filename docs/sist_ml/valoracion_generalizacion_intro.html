Capacidad de Generalización

El criterio definitivo: ¿Sabe el modelo aplicar lo aprendido a lo desconocido?

¿Qué es la generalización?

La capacidad de generalización es el criterio clave para evaluar si un modelo de aprendizaje automático es útil, ya que mide su capacidad para rendir bien en datos no vistos.

No basta con aprender el entrenamiento; el modelo debe captar patrones reales sin depender del ruido. Analizarla implica comparar el rendimiento entre el conjunto de entrenamiento y el de test, entendiendo problemas críticos como el overfitting (sobreajuste) y el underfitting (subajuste).

Overfitting (Sobreajuste)

El modelo "memoriza" el ruido de los datos de entrenamiento. Tiene un error bajísimo en Train pero falla estrepitosamente con datos nuevos (Alta Varianza).

Mala Generalización

Generalización Óptima

El modelo captura la tendencia real. El rendimiento en Train y Test es similar y suficientemente alto. Es el "punto dulce" del aprendizaje.

🗝️ Claves para asegurar la generalización

  • Utilizar un conjunto de Test robusto y representativo.
  • Aplicar técnicas de Regularización si el modelo es muy complejo.
  • Monitorizar la brecha (gap) de error entre entrenamiento y validación.
