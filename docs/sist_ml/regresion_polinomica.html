Regresión Polinómica

Capturando la complejidad más allá de las líneas rectas

¿Por qué necesitamos curvas?

En el mundo real, los datos rara vez siguen una trayectoria perfectamente recta. Imagina la trayectoria de un proyectil, el crecimiento de una bacteria bajo ciertas condiciones o el consumo de combustible de un coche a diferentes velocidades. Estos fenómenos son inherentemente no lineales.

La Regresión Polinómica es una forma de regresión lineal en la que la relación entre la variable independiente $x$ y la variable dependiente $y$ se modela como un polinomio de grado $n$. Aunque el modelo es no lineal respecto a $x$, sigue siendo lineal respecto a los parámetros (los coeficientes que el modelo intenta aprender).

Comparativa: Una línea recta no puede capturar la tendencia curva de los datos.

El poder de las potencias

La magia de este modelo no reside en cambiar el algoritmo de regresión, sino en transformar los datos. En lugar de usar solo $x$ como entrada, creamos nuevas características basadas en el exponente de $x$:

$y = \beta_0 + \beta_1 x + \beta_2 x^2 + \beta_3 x^3 + ... + \beta_n x^n + \epsilon$

A medida que aumentamos el grado del polinomio ($n$), el modelo gana flexibilidad para ajustarse a curvas más complejas. Sin embargo, esto introduce el mayor riesgo de este algoritmo: el sobreajuste.

Grado 1 (Lineal)

Una línea recta. Riesgo de infraajuste (underfitting) si los datos son realmente curvos.

Grado 2-3 (Suave)

Parábolas o curvas en S. Suelen ser excelentes para modelar fenómenos físicos y económicos.

Grado 10+ (Complejo)

La curva serpentea para tocar cada punto. Riesgo extremo de sobreajuste (overfitting).

💻 Implementación con Pipelines

En la práctica, utilizamos PolynomialFeatures para generar las potencias y luego las pasamos a un modelo de LinearRegression. La mejor forma de organizar esto es mediante un Pipeline. 

from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.pipeline import Pipeline
import numpy as np

# Datos de ejemplo (una curva con ruido)
X = np.arange(0, 30).reshape(-1, 1)
y = 0.5 * X**2 - 2*X + 5 + np.random.normal(0, 10, X.shape)

# Crear un Pipeline que:
# 1. Genere las potencias (Grado 2 en este caso)
# 2. Aplique la regresión lineal sobre esas nuevas potencias
model = Pipeline([
    ('poly', PolynomialFeatures(degree=2)),
    ('linear', LinearRegression())
])

# Entrenar
model.fit(X, y)

# Predecir
y_pred = model.predict(X)

print(f"Coeficientes aprendidos: {model.named_steps['linear'].coef_}")
print(f"Intersección: {model.named_steps['linear'].intercept_}")

⚖️ Comparativa: Lineal vs Polinómica

Característica Regresión Lineal Regresión Polinómica
Forma Línea recta (hiperplano) Curva de grado n
Flexibilidad Baja Alta (depende del grado)
Riesgo Principal Infraajuste (Underfitting) Sobreajuste (Overfitting)
Complejidad Mínima Puede explotar con muchos grados y variables

💡 El "Curse of Dimensionality"

Ten mucho cuidado al usar regresión polinómica con múltiples variables de entrada. Si tienes 10 variables y pides un grado 3, el número de características resultantes crecerá exponencialmente (incluyendo interacciones como $x_1 x_2, x_1 x_2 x_3$), lo que puede saturar la memoria y el tiempo de cálculo.
