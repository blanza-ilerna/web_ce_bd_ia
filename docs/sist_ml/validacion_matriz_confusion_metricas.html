6.4.2 Métricas Derivadas y su Interpretación

Más allá de los aciertos: midiendo la calidad y el equilibrio del modelo.

Métricas Globales: Accuracy

La métrica más intuitiva es la Accuracy (Exactitud), que mide la proporción de aciertos totales sobre el total de casos.

Accuracy = (TP + TN) / (TP + TN + FP + FN)

Limitación crítica: Puede ser engañosa en datasets desbalanceados. Si una clase es mayoritaria (ej: 99%), un modelo que prediga siempre esa clase obtendrá un 99% de accuracy sin haber aprendido nada sobre la clase minoritaria.

Calidad vs. Cantidad: Precision y Recall

Precisión (Precision)

Mide la fiabilidad de las predicciones positivas. Responde a: "Cuando el modelo dice positivo, ¿cuántas veces acierta?"

Precision = TP / (TP + FP)

Es vital cuando los Falsos Positivos son costosos (ej: marcar correo legítimo como spam).

Sensibilidad (Recall)

Mide la capacidad de detección. Responde a: "De todos los positivos reales, ¿cuántos detecta el modelo?"

Recall = TP / (TP + FN)

Es crítica cuando los Falsos Negativos son costosos (ej: no detectar una enfermedad grave).

El Compromiso: F1-Score

El F1-Score es la media armónica entre Precision y Recall. A diferencia de un promedio simple, penaliza valores muy bajos en cualquiera de las dos métricas, obligando al modelo a buscar un equilibrio real.

F1 = 2 × (Precision × Recall) / (Precision + Recall)

Interpretación según el Contexto

La elección de la métrica prioritaria depende del impacto del error:

1

Prioridad Recall (Salud / Seguridad)

Preferimos falsas alarmas antes que ignorar un caso real. Aceptamos baja precisión para garantizar que no se escape nada.

2

Prioridad Precision (Productividad / Experiencia)

Preferimos que se escape algún caso antes que molestar al usuario con errores constantes. Evitamos los Falsos Positivos.

Ejemplo Resuelto: Clasificador de Enfermedades

Para aplicar estas fórmulas, usemo el caso de un modelo que analiza a 200 pacientes:

PREDICCIÓN
Pred. Enfermo Pred. Sano
REAL Enfermo 60 (TP) 20 (FN)
Sano 20 (FP) 100 (TN)
Accuracy = (60+100)/200 = 0.80
Precision = 60/(60+20) = 0.75
Recall = 60/(60+20) = 0.75
F1 Score = 2×(0.75×0.75)/1.50 = 0.75

Evaluación Global: Curva ROC y AUC

La Curva ROC representa la relación entre la tasa de verdaderos positivos y la tasa de falsos positivos a distintos umbrales de decisión.

El área bajo esta curva, el AUC (Area Under Curve), resume la capacidad de discriminación del modelo:

  • AUC = 1.0: Modelo perfecto.
  • AUC = 0.5: El modelo no ha aprendido; sus resultados son equivalentes al azar.
  • AUC < 0.5: El modelo está "invertido" (predice lo contrario de lo que ocurre).

Ventaja del AUC

No depende de un umbral específico, lo que permite comparar modelos de forma general antes de decidir el punto de operación final.

Atención

Recuerda que estas métricas son complementarias. Un modelo puede tener un AUC excelente pero una Precision pésima si el umbral elegido no es el adecuado para el problema real.
