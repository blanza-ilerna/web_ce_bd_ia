Ajuste de Hiperparámetros

Configurando la arquitectura del aprendizaje: Buscando el equilibrio perfecto.

¿Qué son los Hiperparámetros?

Los hiperparámetros son configuraciones externas al modelo que definen su comportamiento y estructura antes de que comience el proceso de entrenamiento. A diferencia de los parámetros (que el modelo aprende solo), los hiperparámetros son nuestra herramienta para guiar el aprendizaje.

🌳 Árboles de Decisión

max_depth (profundidad máxima) o min_samples_split.

🌲 Random Forest

n_estimators (número de árboles en el bosque).

🛡️ SVM

C (parámetro de regularización) o el tipo de kernel.

El "tuning" o ajuste fino busca la combinación que maximice la capacidad de generalización. Un ajuste descuidado es una puerta abierta al sobreajuste (overfitting).

1. Búsqueda en Rejilla (Grid Search)

Es el enfoque más tradicional y exhaustivo. Definimos una "rejilla" con valores discretos para cada hiperparámetro y el algoritmo evalúa todas las combinaciones posibles mediante validación cruzada.

✓ Exhaustividad

Garantiza encontrar la mejor combinación dentro de los valores definidos en la rejilla.

✗ Eficiencia

Es computacionalmente muy costoso; a medida que añadimos parámetros, el tiempo crece exponencialmente.

grid_search_example.py 
from sklearn.model_selection import GridSearchCV
from sklearn.tree import DecisionTreeClassifier

# Definimos la rejilla de parámetros
parametros = {
    'max_depth': [2, 4, 6, 8], 
    'criterion': ['gini', 'entropy']
}

modelo = DecisionTreeClassifier(random_state=0)

# Ajuste con CV de 5 carpetas
grid = GridSearchCV(modelo, parametros, cv=5)
grid.fit(X, y)

print("Mejores parámetros encontrados:", grid.best_params_)

2. Búsqueda Aleatoria (Random Search)

En lugar de probar todo, este enfoque selecciona combinaciones aleatorias dentro de un espacio definido. Es mucho más ágil y rápido, permitiendo explorar rangos continuos de valores.

¿Cuándo usarlo?

Es ideal cuando el espacio de búsqueda es inmenso o cuando algunos hiperparámetros influyen mucho más que otros en el rendimiento final.

random_search_example.py 
from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import randint

# Definimos distribuciones de probabilidad en lugar de listas fijas
parametros = {
    'max_depth': randint(2, 10), 
    'min_samples_split': randint(2, 10)
}

# Probamos solo 10 iteraciones aleatorias
rand_search = RandomizedSearchCV(modelo, parametros, n_iter=10, cv=5, random_state=0)
rand_search.fit(X, y)

print("Mejores parámetros aleatorios:", rand_search.best_params_)

3. Optimización Bayesiana

Es el método más inteligente. A diferencia de los anteriores, este proceso aprende de sus propios errores:

Construye un modelo probabilístico (sustituto) que predice qué rendimiento dará una combinación. Utiliza una función de adquisición para decidir si seguir explotando lo que funciona o explorar zonas nuevas.

🚀 Eficiencia Extrema

Converge al óptimo en muchas menos iteraciones que Grid o Random Search.

🧠 Complejidad

Su implementación es más avanzada y requiere mayor comprensión de la lógica interna.
