Funciones de Pérdida

La brújula que guía el aprendizaje de la Red Neuronal evaluando la precisión de sus predicciones.

El Motor de la Optimización

Como hemos mencionado en unidades anteriores, las redes neuronales son capaces de "aprender" porque evalúan la calidad de sus predicciones mediante una función de pérdida (también conocida como función de coste o loss function).

Es, fundamentalmente, la métrica que el optimizador intenta minimizar durante el entrenamiento. La selección adecuada de esta función es absolutamente crítica, ya que condiciona la dirección completa del aprendizaje y la forma en que interpretamos las salidas del modelo. Su elección depende al 100% del tipo de problema matemático a resolver.

Funciones de Pérdida más Comunes

A continuación, desglosamos las tres alternativas más utilizadas dependiendo de la naturaleza de nuestro objetivo predictivo.

📉

Error Cuadrático Medio (MSE) — Regresión

Esta función es la opción predilecta cuando nos enfrentamos a problemas de regresión, es decir, cuando buscamos predecir un valor numérico continuo (ej. precio de una casa, temperatura, ventas).

  • Sensibilidad: Al elevar las diferencias al cuadrado, penaliza de manera muy severa los errores grandes, obligando al modelo a evitarlos.
L = 1/n ∑ (yreal - ypred)2
  • L (Loss): El error total que el modelo intentará minimizar.
  • n: El número total de ejemplos o datos evaluados en este cálculo.
  • ∑ (Sigma): Sumatoria, indica que debemos sumar los errores de todos los n ejemplos.
  • yreal: El valor objetivo o respuesta correcta real en nuestro dataset.
  • ypred: El valor continuo que nuestra red neuronal ha predicho.
mse_example.py 
from sklearn.metrics import mean_squared_error
from sklearn.neural_network import MLPRegressor

# Evaluando predicciones de forma manual
mse_valor = mean_squared_error(y_real, y_predicciones)

# Configurando la función de pérdida en un modelo MLP
modelo_regresion = MLPRegressor(
    hidden_layer_sizes=(100, 50),
    loss='squared_error', # Especificamos MSE
    max_iter=500
)
⚖️

Entropía Cruzada Binaria (BCE) — Clasificación Binaria

Usaremos la Entropía Cruzada Binaria (Binary Cross-Entropy) por defecto en problemas donde solo hay dos clases posibles (ej. spam vs. no spam, enfermo vs. sano).

  • Funcionamiento: Mide matemáticamente la "distancia" entre la probabilidad predicha p (un valor de 0 a 1) y la etiqueta real y (que es 0 o 1).
L = - 1/n ∑ [y · log(p) + (1 - y) · log(1 - p)]
  • L (Loss): El valor de la Entropía Cruzada Binaria.
  • n: Número de elementos en el lote de datos analizado.
  • y: La clase o etiqueta real, que obligatoriamente será 0 o 1.
  • p: La probabilidad (entre 0 y 1) predicha por la red de pertenecer a la clase 1.
  • log: Logaritmo natural, encargado de castigar severamente las predicciones que están muy seguras pero son incorrectas.
bce_example.py 
from sklearn.metrics import log_loss
from sklearn.neural_network import MLPClassifier

# Calculando la pérdida a partir de probabilidades predichas
# (y_probabilidades debe contener la probabilidad de la clase positiva)
bce_valor = log_loss(y_real, y_probabilidades)

# Configurando un clasificador MLP binario
modelo_binario = MLPClassifier(
    hidden_layer_sizes=(64, 32),
    activation='logistic', # Utiliza sigmoide internamente para clasificación binaria
    max_iter=500
) # Sklearn usa log-loss automáticamente para clasificación iterativa
📊

Entropía Cruzada Categórica (CCE) — Clasificación Multiclase

Esta función es el estándar absoluto cuando el problema implica tres o más clases excluyentes (como clasificar imágenes de perros, gatos y pájaros).

  • Distribuciones de Probabilidad: Compara la distribución de probabilidad generada por la última capa de la red (típicamente una capa Softmax) con la distribución real u objetivo (representada como un 1 en la clase correcta y 0 en todas las demás, conocido como One-Hot Encoding).
L = - ∑ [y · log(p)]
  • L (Loss): El valor de la Entropía Cruzada Categórica.
  • ∑ (Sigma): Aquí se suma a través de todas las clases posibles para un solo ejemplo.
  • y: El valor real codificado en One-Hot (ej. 1 si es un perro, 0 para gato o pájaro).
  • p: La probabilidad que la red asigna a esa clase específica. Dado que y será 0 para las clases incorrectas, la fórmula se centra únicamente en maximizar la probabilidad p de la clase correcta (donde y=1).
cce_example.py 
from sklearn.metrics import log_loss
from sklearn.neural_network import MLPClassifier

# En clasificación multiclase, pasamos un array 2D con las 
# probabilidades para cada clase
cce_valor = log_loss(y_real, y_probabilidades_multiclase)

# Configurando un clasificador MLP multiclase
modelo_multiclase = MLPClassifier(
    hidden_layer_sizes=(128, 64),
    max_iter=1000
) # Sklearn maneja automáticamente la entropía cruzada categórica internamente

En Resumen: ¿Cuándo usar cada una?

📉 Regresión

Utiliza el Error Cuadrático Medio (MSE) cuando tu objetivo sea predecir valores numéricos continuos. Penaliza severamente aquellos errores que se alejan de la realidad por estar elevados al cuadrado.

⚖️ Clas. Binaria

La Entropía Cruzada Binaria (BCE) es el estándar inamovible en problemas estadísticos donde sólo hay dos posibles desenlaces excluyentes (0 o 1, Sí o No).

📊 Clas. Multiclase

Aplica la Entropía Cruzada Categórica (CCE) para diferenciar entre tres o más categorías u opciones cuando el objetivo de predicción sólo puede estar etiquetado en una.
