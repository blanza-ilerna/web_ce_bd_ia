Codificación de Variables Nominales

Eliminando el sesgo de orden en categorías puras.

¿Por qué no usar números correlativos?

A diferencia de las variables ordinales, las nominales (como marcas de coches, colores o países) no tienen una jerarquía lógica. Si asignamos números arbitrarios (Seat=1, Citroën=2, Opel=3), el modelo podría interpretar matemáticamente que un "Citroën" es la media entre un "Seat" y un "Opel", lo cual es una conclusión absurda.

⚠️ El peligro de la aritmética falsa

Asignar valores 1, 2, 3... a categorías sin orden induce al modelo a buscar relaciones espaciales y de distancia que no existen, degradando la lógica del aprendizaje.

Cómo funciona el One-Hot Encoding

El término "One-Hot" proviene del diseño de circuitos digitales, donde una señal i-ésima está en estado alto (hot/1) mientras las demás están en bajo (cold/0). En Machine Learning, este método crea un vector binario para cada categoría.

Mecánica de la exclusividad

Si tenemos 3 categorías (Rojo, Verde, Azul):

  • Rojo: Se representa como [1, 0, 0]
  • Verde: Se representa como [0, 1, 0]
  • Azul: Se representa como [0, 0, 1]

Esta estructura garantiza que el modelo entienda que las categorías son mutuamente excluyentes. Un dato no puede ser "un poco rojo y un poco azul" en este esquema; es una cosa o la otra.

Implementación Práctica

Con Pandas, podemos generar estas columnas "dummy" de forma extremadamente sencilla:

one_hot_encoding.py 
import pandas as pd

# Suponiendo que 'Color' es nuestra columna nominal
df_encoded = pd.get_dummies(df, columns=['Color'])

# Para evitar la multicolinealidad (Trampa de Variables Dummy)
df_encoded_safe = pd.get_dummies(df, columns=['Color'], drop_first=True)

Desventajas a considerar

Maldición de la Dimensionalidad

Si la variable tiene muchas categorías únicas (ej. Códigos Postales), generaremos cientos de columnas, disparando el uso de memoria y ralentizando el cálculo.

Trampa de Variables Dummy

Introduce multicolinealidad perfecta (si sabemos que no es negro ni azul, obligatoriamente es rojo). Se soluciona eliminando la primera columna (drop_first=True).

Técnicas para Alta Cardinalidad

Cuando nos enfrentamos a variables con cientos o miles de categorías (como modelos de coches específicos o ciudades), el One-Hot Encoding se vuelve inmanejable. Aquí es donde brillan otras estrategias:

📦 Binary Encoding (Codificación Binaria)

Es una de las técnicas más eficientes para reducir la dimensionalidad. Funciona convirtiendo las categorías en números y luego en bits distribuidos en columnas.

binary_encoding.py 
import category_encoders as ce

# Almacenamiento eficiente: 100 categorías en solo 7 columnas
encoder = ce.BinaryEncoder(cols=['Ciudad'])
df_binary = encoder.fit_transform(df)

🎯 Target Encoding

Sustituye cada categoría por la media de la variable objetivo. Es extremadamente potente pero requiere cuidado con el overfitting.

target_encoding.py 
# Mapeo estadístico basado en el objetivo
means = df.groupby('Marca')['Precio'].mean()
df['Marca_Target'] = df['Marca'].map(means)

🔏 Hashing Encoding

Utiliza una función de hash para mapear categorías a un espacio de columnas fijo. Excelente para datos que crecen constantemente.

hashing_encoding.py 
import category_encoders as ce
hasher = ce.HashingEncoder(n_components=8)
df_hash = hasher.fit_transform(df['Categoria'])

📊 Frequency / Count Encoding

Mapea cada categoría con su abundancia estadística dentro del dataset.

frequency_encoding.py 
# Frecuencias normalizadas (0 a 1)
frecuencias = df['Color'].value_counts(normalize=True)
df['Color_Freq'] = df['Color'].map(frecuencias)

🚀 Resumen de decisión rápida

Situación Técnica Recomendada
< 10 categoríasOne-Hot Encoding (Estándar)
10 - 50 categoríasBinary Encoding
Categorías masivas (>100)Hashing o Frequency
Modelos lineales / RegresiónCuidado con One-Hot (usar drop_first)
