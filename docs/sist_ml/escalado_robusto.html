Escalado Robusto (RobustScaler)

Blindando el preprocesamiento frente a valores atípicos (outliers).

¿Qué es el RobustScaler?

El Escalado Robusto es una técnica de preprocesamiento diseñada específicamente para minimizar la influencia de los outliers (valores atípicos) en el proceso de escalado.

A diferencia de otros métodos, el RobustScaler no utiliza ni la media ni el máximo/mínimo, ya que estas estadísticas son extremadamente sensibles a valores extremos:

❌ MinMaxScaler

Sensible a los extremos máximos y mínimos que definen el rango de compresión.

❌ StandardScaler

La media y la desviación estándar se ven arrastradas por valores muy alejados.

La Fórmula del Escalado Robusto

Este método centra los datos utilizando la mediana y los escala basándose en el Rango Intercuartílico (IQR). Al basarse en la posición de los datos y no en su valor aritmético, las anomalías dejan de dominar la transformación.

X' =
X - Mediana IQR

Conceptualmente, la fórmula es muy similar a la de MinMaxScaler, pero sustituye los valores extremos (Mín/Máx) por cuartiles para cubrirnos ante ruido estadístico.

Mediana

Valor central (Q₂ / Percentil 50)

IQR

Cuartil 3 - Cuartil 1 (Q₃ - Q₁)

Comprendiendo el Rango Intercuartílico (IQR)

El Rango Intercuartílico (IQR) es una medida de dispersión estadística que representa la distancia entre el primer y el tercer cuartil. Es decir, describe dónde se concentra el 50% central de los datos.

IQR = Q3 - Q1

¿Por qué es clave para la robustez?

Al ignorar el 25% inferior y el 25% superior de la distribución (donde suelen esconderse los valores atípicos), el IQR proporciona una escala mucho más estable que la desviación estándar o el rango absoluto (Máx - Mín).

En términos de percentiles:

Q₁

Percentil 25 (Límite inferior del 50% central)

Q₃

Percentil 75 (Límite superior del 50% central)

Implementación con Scikit-Learn

Utilizamos el estimador RobustScaler del módulo preprocessing de Sklearn. Es ideal para datasets con sensores ruidosos o datos financieros con picos atípicos.

robust_scaling.py 
from sklearn.preprocessing import RobustScaler
import pandas as pd

# Supongamos un dataset con ingresos y algunos valores extremos
data = {'ingresos': [25000, 30000, 28000, 32000, 1000000]} # 1M es un outlier
df = pd.DataFrame(data)

# Inicializamos el escalador
# quantile_range=(25.0, 75.0) es el valor por defecto (IQR)
scaler = RobustScaler()

# Ajustamos y transformamos
# La mediana de los ingresos será el nuevo 0
df['ingresos_escalados'] = scaler.fit_transform(df[['ingresos']])

print(df)

Reflexión Final: ¿Cuál elegir?

Seleccionar el método adecuado exige comprender tanto la naturaleza de los datos como las características del modelo que se va a emplear. Mientras que Min-Max es útil para redes neuronales y algoritmos que requieren rangos fijos, el RobustScaler es el salvavidas cuando nuestros datos presentan anomalías que no queremos (o podemos) eliminar del proceso.

En conjunto, estas técnicas representan un paso crítico para garantizar la solidez y fiabilidad del proceso de aprendizaje automático.
