Selección de Características

Optimizando la eficiencia: Menos es más en la ingeniería de variables.

Calidad sobre Cantidad

Aunque solemos pensar que disponer de más datos es siempre beneficioso, en el aprendizaje automático esta máxima no se aplica de igual forma a las características (dimensiones).

Los datasets modernos pueden contener cientos o miles de variables. Sin embargo, procesar variables irrelevantes o redundantes conlleva graves problemas:

🚫 Ruido Estadístico

Las variables irrelevantes degradan la capacidad de generalización del modelo.

⚠️ Sobreajuste (Overfitting)

Demasiadas dimensiones facilitan que el modelo memorice casos particulares en lugar de patrones.

⏳ Coste Computacional

Más características significan entrenamientos más lentos y mayor consumo de memoria.

1. Métodos de Filtro (Filter Methods)

Estos métodos actúan como un **paso de preprocesamiento** independiente. Evalúan la relevancia de cada variable basándose exclusivamente en sus propiedades estadísticas intrínsecas, normalmente comparándolas con el target.

Técnicas Comunes

Pruebas de correlación de Pearson, Chi-cuadrado (variables categóricas) o ANOVA. Se asigna una puntuación y se filtran aquellas que no superan un umbral.

✓ Ventajas

  • Extremadamente rápidos y eficientes.
  • Independientes del modelo final (agnósticos).

✗ Desventajas

  • Ignoran por completo las interacciones entre variables.
metodo_filtro.py 
from sklearn.feature_selection import SelectKBest, chi2
from sklearn.datasets import load_breast_cancer

# Carga de datos
X, y = load_breast_cancer(return_X_y=True)

# Seleccionamos las 10 mejores características usando Chi-cuadrado
selector = SelectKBest(chi2, k=10)
X_reducido = selector.fit_transform(X, y)

print("Características originales:", X.shape[1])
print("Características seleccionadas:", X_reducido.shape[1])

2. Métodos de Envoltura (Wrapper Methods)

A diferencia de los filtros, estos métodos "envuelven" la selección alrededor del algoritmo específico de Machine Learning. El proceso se convierte en un **problema de búsqueda**.

Se prueban distintos subconjuntos de variables entrenando el modelo real y evaluando su rendimiento (normalmente mediante validación cruzada) para decidir cuáles son las más efectivas.

RFE (Recursive Feature Elimination)

Elimina iterativamente las variables menos importantes hasta alcanzar el número deseado.

Forward Selection

Empieza sin variables y añade paso a paso la que más mejora el rendimiento.

metodo_envoltura.py 
from sklearn.feature_selection import RFE
from sklearn.linear_model import LogisticRegression
from sklearn.datasets import load_breast_cancer

X, y = load_breast_cancer(return_X_y=True)

# Usamos Regresión Logística para evaluar la importancia
modelo = LogisticRegression(max_iter=5000)
selector = RFE(modelo, n_features_to_select=8)

# Ajustamos el proceso de selección
selector.fit(X, y)

# Resultados
print("Máscara de selección:", selector.support_)
print("Ranking de importancia:", selector.ranking_)

Hacia un flujo de trabajo optimizado

La selección de características no solo busca el mejor rendimiento, sino la interpretabilidad. Un modelo con menos variables es más fácil de depurar, más rápido de ejecutar en producción y, sobre todo, mucho menos propenso a aprender ruido aleatorio del dataset.
