Ejemplo de Red Convolucional

Desarrollando un modelo de Visión Artificial avanzado para leer y reconocer dígitos escritos a mano.

Clasificando el Dataset MNIST

Vamos ahora a desarrollar un ejemplo práctico completo de la aplicación de una arquitectura de Red Neuronal Convolucional (CNN) a un problema visual real. En concreto, vamos a entrenar desde cero un cerebro artificial capaz de clasificar observaciones visuales del famoso conjunto de datos MNIST.

El dataset MNIST es un conjunto de datos público que agrupa miles de pequeñas imágenes (con dimensiones de 28x28 píxeles) en escala de grises de dígitos manuscritos del 0 al 9, siendo considerado a día de hoy el "Hola Mundo" del Deep Learning visual.

⚠️ Atención: Entorno de Ejecución

Para que el entrenamiento interactivo no demore muchísimo tiempo, se recomienda fuertemente ejecutar este script en plataformas como Google Colab que brinden soporte de procesamiento por hardware dedicado (GPU/TPU).

1. Importación de Librerías

Comenzamos por invocar el ecosistema de TensorFlow/Keras, y en especial, traemos las capas necesarias para construir la topología visual.

1_imports.py 
import tensorflow as tf
from tensorflow.keras import layers, models
from tensorflow.keras.datasets import mnist
from tensorflow.keras.utils import to_categorical

2. Carga y Normalización del Espacio Visual

A diferencia de nuestra anterior red predictiva basada en características comunes (como la longitud de un pétalo), las matrices de un píxel, al estar en blanco y negro puros, miden la luminosidad enviando valores en un espectro que va desde el número 0 (negro puro) hasta el 255 (blanco puro).

Nuestro primer procedimiento técnico consistirá en normalizar todos estos rangos y encogerlos obligatoriamente dentro de un margen microscópico numérico [0.0, 1.0]. Esta práctica acelerará brutalmente el ritmo de la convergencia.

2_load_normalize.py 
# Cargar y preparar los datos nativos
(X_train, y_train), (X_test, y_test) = mnist.load_data()

# Normalizar valores de píxeles dividiendo entre el valor lumínico máximo
X_train = X_train.astype('float32') / 255.0
X_test = X_test.astype('float32') / 255.0

Finalmente procedemos a redimensionar (reshape) los datos para amoldarlos explícitamente a que cumplan el requerimiento CNN de poseer un indicador del Canal de color. Al ser Blanco y Negro, definimos 1 capa de profundidad de canal (1). Y naturalmente codificamos One-Hot la etiqueta de salida porque hay 10 clases de resoluciones posibles correspondientes a los 10 dígitos ordinales.

3_reshape_targets.py 
# Redimensionar para incluir el canal de color (-1 es automatismo total de Keras, 28x28 alto x ancho, 1 Canal Grises)
X_train = X_train.reshape(-1, 28, 28, 1)
X_test = X_test.reshape(-1, 28, 28, 1)

# Codificar las etiquetas como vectores one-hot identificadores para 10 clases temporales exclusivas
y_train = to_categorical(y_train, 10)
y_test = to_categorical(y_test, 10)

3. Ensamblando y Trazando la Red Convolucional

Del mismo modo conceptual visto en la práctica de la unidad pasada, empilaremos ordenadamente módulos dentro de un armazón Sequential:

  • Capa Principal Conv2D: Aquí nace toda la magia de extracción. Mapea la silueta enviando simultáneamente 32 filtros matemáticos cuadriculados de 3x3 píxeles (ventana) a barrer por todo el lienzo en búsqueda natural de curvas o rectas microscópicas (patrones locales visuales). Adhiere al final un filtro de excitación y no linealidad (ReLU). input_shape define firmemente nuestra inconfundible estampa original 28x28 de 1 canal.
  • Sub-capa MaxPooling2D: Condensador que recibe temporalmente los 32 fotogramas (mapas) provenientes de su homóloga convolucional y la fuerza a dividirse comprimiendo el lienzo y preservando casi la máxima información, todo mediante la observación diminuta 2x2.
  • Capas Profundas (64 filtros): Agregamos un segundo bloque extractor *Conv2D* combinado a posteriori con otro sumidero volumétrico *MaxPooling*. Como te percatarás, al profundizar internamente la red, incrementamos la búsqueda de hallazgos matemáticos abstractos elevando de 32 hasta 64 filtros. Aprenderá a fusionar y combinar lo encontrado por los primeros 32 para lograr trazos gruesos reconocibles u ojivas.
  • Capa Extructuradora Flatten: Bloque que actúa como puente, cogiendo lo que actualmente es puré estructural multidimensional (mapas) y enrollándolos físicamente formando por gravedad un enorme tubo Vector lineal unidireccional.
  • Clasificadores Terminales (Dense): Finalmente conectará todo ese vector puritano (Output 1600) y se lo arrojará a unas capas completamente funcionales ordinarias con su activador lineal interno 64 (ReLU), donde el peso será dictaminada la certeza bajo la batuta del activador final estadístico de multiclase 10 (Categorical Softmax).

4_cnn_architecture.py 
# Definir la grandiosa topología abstracta de la CNN visual
model = models.Sequential([
    layers.Conv2D(32, (3,3), activation='relu', input_shape=(28,28,1)),
    layers.MaxPooling2D((2,2)),
    layers.Conv2D(64, (3,3), activation='relu'),
    layers.MaxPooling2D((2,2)),
    layers.Flatten(),
    layers.Dense(64, activation='relu'),
    layers.Dense(10, activation='softmax')
])

4. Compilación, Entrenamiento y Evaluación

Cerramos el ciclo de Deep Learning pidiéndole al entorno que compile todos los engranajes empleando el optimizador maestro Adam, ajustando luego la red iterativamente pasándole trozos de memoria con una densidad batch_size fijada estratégicamente en 64 para acelerar los cómputos. Y finalmente validamos la bondad obtenida evaluando los datos oscurecidos inicialmente.

5_train_evaluate.py 
# Compilar fijando motor matemático y métrica final
model.compile(optimizer='adam',
              loss='categorical_crossentropy',
              metrics=['accuracy'])

# Entrenarlo duramente dando 5 repasos intensivos (epochs) en la GPU
history = model.fit(X_train, y_train, epochs=5, batch_size=64, validation_split=0.1, verbose=1)

# Juzgar su desempeño sobre el lote aislado puro
test_loss, test_acc = model.evaluate(X_test, y_test, verbose=0)
print(f"\nExactitud en el conjunto de prueba MNIST: {test_acc:.3f}")

Código Completo del Clasificador CNN

Si la configuración fue exitosa, este cerebro debió promediar alrededor de una apabullante fiabilidad técnica en métricas que ronda en torno a un 99% de Accuracy sobre información externa pura de testing. ¡Una verdadera maravilla matemática moderna!

mnist_cnn_clasificador.py 
import tensorflow as tf
from tensorflow.keras import layers, models
from tensorflow.keras.datasets import mnist
from tensorflow.keras.utils import to_categorical

# 1. Cargar y preparar los datos
(X_train, y_train), (X_test, y_test) = mnist.load_data()

# 2. Normalizar valores de píxeles (0–255) al rango [0,1]
X_train = X_train.astype('float32') / 255.0
X_test = X_test.astype('float32') / 255.0

# 3. Redimensionar para incluir el canal (1 para imágenes en gris)
X_train = X_train.reshape(-1, 28, 28, 1)
X_test = X_test.reshape(-1, 28, 28, 1)

# 4. Codificar las etiquetas como vectores one-hot
y_train = to_categorical(y_train, 10)
y_test = to_categorical(y_test, 10)

# 5. Definir la poderosa arquitectura de la CNN de Extracción
model = models.Sequential([
    layers.Conv2D(32, (3,3), activation='relu', input_shape=(28,28,1)),
    layers.MaxPooling2D((2,2)),
    layers.Conv2D(64, (3,3), activation='relu'),
    layers.MaxPooling2D((2,2)),
    layers.Flatten(),
    layers.Dense(64, activation='relu'),
    layers.Dense(10, activation='softmax')
])

# 6. Compilar el modelo enlazando las funciones vitales
model.compile(optimizer='adam',
              loss='categorical_crossentropy',
              metrics=['accuracy'])

# 7. Iniciar el Entrenamiento pesando en tarjetas Gráficas recomendadas
history = model.fit(X_train, y_train, epochs=5, batch_size=64, validation_split=0.1, verbose=1)

# 8. Comprobar validez resolutiva final 
test_loss, test_acc = model.evaluate(X_test, y_test, verbose=0)
print(f"\nExactitud en el conjunto de prueba MNIST: {test_acc:.3f}")
