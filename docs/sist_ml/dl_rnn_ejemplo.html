Ejemplo de una RNN

Creando un analizador de sentimientos usando texto secuencial y una estructura LSTM avanzada.

Clasificando Reseñas Literales en IMDB

Vamos ahora a desarrollar un ejemplo práctico fascinante sobre la aplicación directa de una arquitectura de Red Neuronal Recurrente (RNN). En concreto, vamos a usar una estructura evolucionada denominada LSTM (Long Short-Term Memory) para llevar a cabo una de las tareas estrella del Procesamiento de Lenguaje Natural (NLP): el análisis de sentimientos.

Nuestra misión será entrenar una red capaz de leer texto escrito (clasificar sentencias) y dictaminar si expresan un tono emocional positivo o negativo, usando como maestro el monumental conjunto de datos público IMDB, que acumula hasta 50.000 reseñas genuinas de películas etiquetadas de forma binaria.

⚠️ Atención: Naturaleza Secuencial

Recuerda que a diferencia de una foto estática (CNN), una frase no existe de forma simultánea. Se pronuncia secuencialmente, una palabra detrás de otra, y el significado de la palabra actual depende estrictamente del contexto aportado por la palabra anterior. ¡Ese es el terreno natural de las RNN!

1. Importando nuestro arsenal analítico

Lógicamente, requerimos importar herramientas especializadas en la manipulación de secuencias que no habíamos empleado hasta ahora.

1_imports.py 
import tensorflow as tf
from tensorflow.keras.datasets import imdb
from tensorflow.keras.preprocessing.sequence import pad_sequences
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Embedding, LSTM, Dense, Dropout

2. Carga y Uniformidad de Secuencias Tildadas

El conjunto de entrada tiene un reto singular: no todas las personas escriben reseñas del mismo tamaño. Una crítica puede ser solo "mala película" (2 palabras) mientras que otra puede extenderse a miles. Una matriz matemática exige estructura uniforme.

Utilizamos dos técnicas clave: Primero filtramos el diccionario reteniendo solo el Top 1.000 de palabras más influyentes (num_words). Y segundo, la estrella de la sección: forzamos matemáticamente a que todas las oraciones midan exactamente lo mismo mediante pad_sequences (rellenando con ceros los huecos en reseñas cortas o podando bruscamente las que excedan una longitud máxima prefijada).

2_load_pad_nlp.py 
# Instanciar Dataset considerando únicamente las 1000 palabras más frecuentes en la jerga
num_palabras = 1000 
(X_train, y_train), (X_test, y_test) = imdb.load_data(num_words=num_palabras)

print("Tamaño del conjunto de entrenamiento:", len(X_train))
print("Ejemplo de secuencia codificada (índices numéricos de diccionario):\n", X_train[0][:20])

# Preprocesamiento: Padding (Relleno y Truncado uniformizador)
max_longitud = 100 # Fijamos tajantemente: Todas las secuencias medirán exactamente 100 tokens

X_train = pad_sequences(X_train, maxlen=max_longitud, padding='post', truncating='post')
X_test = pad_sequences(X_test, maxlen=max_longitud, padding='post', truncating='post')

3. Articulando la Memoria y Comprensión LSTM

Pasemos de los tensores base a modelar el procesador del lenguaje dentro de nuestra jaula Sequential:

  • Traductor Cuántico (Embedding): Los cerebros artificiales no asimilan texto desnudo ("casa"). Transforman las letras a identificadores de diccionario ("casa" -> 73), pero para "comprender el semantismo" (es decir, que casa, edificio y hotel están matemáticamente emparentados por su significado), la capa Embedding enhebra y entrena vectorialmente esta asociación ubicando la palabra en una inmensa galaxia matemática de 64 dimensiones espaciales o output_dim. Cuanto mayor es esta dimensión, más rico y contextual es su registro asociativo cerebral.
  • El Núcleo Memorable (LSTM): La pieza central recurrente. Parametrizada con 128 "neuronas de memoria", se dedica a transitar sobre esas secuencias vectoriales (leyendo secuencialmente tu reseña). Su variable clave es return_sequences=False, que indica amablemente a la capa que sólo nos escupa el desenlace absoluto y final del dictamen de toda la secuencia entera tras haber leído toda la amalgama de texto al unísono, y no que vaya publicando valoraciones intermitentes subyacentes.
  • Amortiguador (Dropout): Inyectamos un severo castigo apagando deliberadamente la sinápsis de un 20% de su estructura neuronal aleatoria durante el entrenamiento para evitar que experimente Overfitting, previniendo que memorice sentencias específicas por puro músculo.
  • Sentenciador Total (Dense): Como remate a toda este armatoste, como estamos analizando un dictamen categórico binario ("Opinión Buena o Mala"), finalizamos el entubado inyectándolo dentro de una sola neurona aislada que evaluará la salida mediante la función sigmoide, arrojando su sentencia en forma probabilística [0.0 a 1.0].

3_rnn_architecture.py 
# Construcción arquitectónica del modelo lingüístico LSTM
model = Sequential([
    Embedding(input_dim=num_palabras, output_dim=64, input_length=max_longitud),
    LSTM(128, return_sequences=False),
    Dropout(0.2),
    Dense(1, activation='sigmoid') # Salida binaria terminal: (0 = negativo / 1 = positivo)
])

4. Certificación y Compilación del Analista

Al ser un escenario de polaridad enfrentada, usamos una función correctiva binary_crossentropy para castigar matemáticamente al motor central si en algo tan obvio dictamina falsamente una probabilidad mal encaminada, midiendo mediante su precisión neta en puros aciertos.

4_train_evaluate.py 
# Compilación penalizatoria de tipo Binomial
model.compile(optimizer='adam', loss='binary_crossentropy', metrics=['accuracy'])

# Entrenamiento agresivo en la CPU/GPU asimilando sintaxis de reseñas
history = model.fit(
    X_train, y_train,
    epochs=4,
    batch_size=128,
    validation_split=0.2,
    verbose=1
)

# Validación frente al texto que las neuronas jamás han visto antes
loss, accuracy = model.evaluate(X_test, y_test, verbose=0)
print(f"\nExactitud del modelo en el conjunto de prueba (Lentamente Leyendo): {accuracy:.3f}")

Código Completo de la Red Recurrente (RNN)

Con sólo las 1.000 palabras más repetidas y unos rápidos repasos superficiales limitados a 100 espacios dimensionales, ya este modelo rascó de su contexto secuencial **en torno al 80% de Accuracy inquebrantable en datos de testeo**. Un poder de contextualización abrumador para lidiar contra la subjetividad pura humana en un bloque simple de código.

imdb_rnn_sentimientos_full.py 
import tensorflow as tf
from tensorflow.keras.datasets import imdb
from tensorflow.keras.preprocessing.sequence import pad_sequences
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Embedding, LSTM, Dense, Dropout

# 1. Cargar el dataset IMDB
num_palabras = 1000 # Se considerarán las 1000 palabras más frecuentes
(X_train, y_train), (X_test, y_test) = imdb.load_data(num_words=num_palabras)

# 2. Preprocesamiento: Padding, normalización de longitud estricta a 100 campos
max_longitud = 100
X_train = pad_sequences(X_train, maxlen=max_longitud, padding='post', truncating='post')
X_test = pad_sequences(X_test, maxlen=max_longitud, padding='post', truncating='post')

# 3. Ensamblaje secuencial maestro basado en memoria lingüística de tipo RNN
model = Sequential([
    # Traductor numérico dimensional a vecindad semántica generalizada
    Embedding(input_dim=num_palabras, output_dim=64, input_length=max_longitud),
    # Controlador neurálgico maestro
    LSTM(128, return_sequences=False),
    # Castigo regulador del autoengaño empírico
    Dropout(0.2),
    # Sigmoide Final binario
    Dense(1, activation='sigmoid')
])

# 4. Formulación abstracta en el compilador del ecosistema
model.compile(optimizer='adam', loss='binary_crossentropy', metrics=['accuracy'])

# 5. Iteración formativa en CPU y acopio de fallos graduales
history = model.fit(
    X_train, y_train,
    epochs=4,
    batch_size=128,
    validation_split=0.2,
    verbose=1
)

# 6. Refrendo de efectividad con datos de testeo ocultos
loss, accuracy = model.evaluate(X_test, y_test, verbose=0)
print(f"\nExactitud del modelo predictivo en el conjunto oscuro de prueba: {accuracy:.3f}")
