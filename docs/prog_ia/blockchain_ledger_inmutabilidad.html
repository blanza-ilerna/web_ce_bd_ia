Inmutabilidad del Registro 🛡️

La permanencia y seguridad de los datos en sistemas distribuidos.

¿Qué es la Inmutabilidad?

La inmutabilidad garantiza que, una vez que una transacción ha sido validada por la red e incorporada al libro mayor, resulta imposible modificarla o eliminarla sin que el sistema detecte la ruptura de las reglas.

Esta propiedad no implica que no se puedan corregir errores; simplemente significa que el historial no se borra. Si se comete un error, se debe emitir una nueva transacción compensatoria, manteniendo el rastro completo de lo sucedido.

El Mecanismo de Seguridad: El Hash

La inmutabilidad se apoya en la criptografía para encadenar la información de forma indestructible:

📦 Estructura de Bloque

Cada bloque agrupa transacciones, marcas de tiempo y, lo más importante, el Hash del bloque anterior.

🔗 Efecto Dominó

Si alguien altera un dato antiguo, su Hash cambia radicalmente. Esto invalida el enlace con el bloque siguiente, rompiendo la cadena.

Inviabilidad del Ataque

Para manipular el pasado, un atacante tendría que rehacer todo el trabajo computacional posterior o controlar la mayoría de la red, algo que es computacionalmente inviable en redes bien diseñadas.

Ejemplo Práctico: Puntos de Clase 📋

Imaginemos que queremos registrar "puntos de participación" en el aula:

📁 Sistema Tradicional (Excel)

  • El profesor guarda el archivo en su PC.
  • Si el archivo se pierde o se altera, no hay forma de verificar el historial real.
  • Dependencia total de una autoridad central.

🌐 Sistema DLT (Distribuido)

  • Cada alumno tiene una copia idéntica del registro en su dispositivo.
  • Cada punto nuevo se difunde y valida en tiempo real por todos.
  • Cualquier intento de manipulación es detectado por la red.

Este ejemplo ilustra cómo la distribución y la inmutabilidad crean un entorno de confianza compartida, donde el historial de eventos permanece íntegro y auditable por todos los participantes.
