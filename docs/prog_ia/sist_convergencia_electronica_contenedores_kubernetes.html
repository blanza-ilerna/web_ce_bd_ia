Kubernetes (k8s)

Orquestación inteligente para ecosistemas de contenedores masivos ☸️

¿Por qué necesitamos Orquestación?

Cuando una infraestructura digital crece y pasa de gestionar unos pocos contenedores a manejar decenas o cientos de ellos, el control manual se vuelve imposible. Surge la necesidad de una herramienta que automatice decisiones críticas:

📍 Planificación

Decide en qué máquinas físicas debe ejecutarse cada contenedor basándose en la carga.

🩹 Auto-curación

Detecta contenedores caídos y los vuelve a levantar automáticamente.

⚖️ Balanceo

Reparte el tráfico de forma equitativa entre varias réplicas de un mismo servicio.

🔄 Actualización

Gestiona despliegues de nuevas versiones sin interrumpir el servicio al usuario.

Definición de Kubernetes

Kubernetes ☸️ es una plataforma de código abierto diseñada para automatizar el despliegue, el escalado y la gestión de aplicaciones en contenedores. Se ha convertido en el estándar de facto para la orquestación en entornos Cloud Native.

Elementos Clave del Ecosistema

Para entender cómo funciona Kubernetes, debemos conocer sus unidades fundamentales de organización:

🌐 Cluster de Kubernetes

Es el conjunto total de recursos: una agrupación de máquinas (nodos) que trabajan como una sola unidad.

🖥️ Nodo

Cada una de las máquinas (físicas o virtuales) que forman el cluster y ejecutan los contenedores.

📦 Pod

La unidad mínima de despliegue. Puede contener uno o varios contenedores muy relacionados que comparten red y almacenamiento.

📜 Deployment

Define el estado deseado: cuántas réplicas de un Pod deben existir y cómo deben actualizarse.

🔌 Service

Proporciona un punto de acceso estable (IP virtual o nombre DNS) que reparte el tráfico entre los Pods dinámicos.

Ejemplo: Sistema Distribuido de IA

Imaginemos una aplicación de recomendaciones personalizadas compuesta por tres microservicios:

  • API Gateway (Contenedor A): Recibe las peticiones de los usuarios. En Kubernetes se define como un Deployment con varias réplicas para soportar picos de tráfico.
  • Modelo IA (Contenedor B): Ejecuta el algoritmo de recomendación. Kubernetes aplica reglas de Autoscaling para añadir réplicas si la CPU sube demasiado.
  • Base de Datos (Contenedor C): Almacena preferencias de usuario. Kubernetes asegura que el servicio sea accesible de forma persistente para los otros microservicios.

Si un nodo del hardware falla, Kubernetes reprograma los Pods de la API y el Modelo IA en otro nodo sano de forma transparente, manteniendo la aplicación siempre activa.

Papel de Docker y Kubernetes

Es común confundirlos, pero trabajan en capas distintas de la infraestructura:

Capa Herramienta Función principal
Contenedor Docker Construir y ejecutar imágenes de contenedor de forma aislada.
Orquestación Kubernetes Gestionar y coordinar miles de contenedores en múltiples nodos.

Puntos Clave

  • Fomenta la orquestación automatizada de contenedores.
  • Utiliza conceptos de Cluster, Nodo, Pod, Deployment y Servicio.
  • Las funciones vitales son el escalado automático y la auto-recuperación.

Atención: Curva de Aprendizaje

Kubernetes introduce una gran complejidad técnica. Solo tiene sentido cuando el volumen de servicios es elevado o se requiere una alta disponibilidad crítica. Para proyectos pequeños o aplicaciones monolíticas, su uso puede resultar excesivo y costoso de mantener.
