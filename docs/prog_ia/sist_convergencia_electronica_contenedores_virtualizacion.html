Virtualización de Hardware

La capacidad de ejecutar múltiples mundos lógicos sobre una única realidad física

¿Qué es la Virtualización?

La virtualización es la técnica que permite fragmentar un ordenador físico en varias "máquinas lógicas". Cada una de estas entidades, denominadas Máquinas Virtuales (VM), opera de forma totalmente independiente, disponiendo de su propio sistema operativo, asignación de memoria y aplicaciones específicas.

El Motor: El Hipervisor

El componente clave de este sistema es el Hipervisor (Virtual Machine Monitor o VMM). Su función principal es actuar como un árbitro que reparte dinámicamente los recursos físicos del hardware (CPU, RAM, disco y red) entre las distintas VMs que se ejecutan sobre él.

Tipos de Hipervisores

Dependiendo de dónde se instale la capa de gestión, distinguimos dos arquitecturas principales:

🛠️ Tipo 1 (Bare Metal)

Se instala directamente sobre el hardware físico del servidor. Es la opción más eficiente para centros de datos.

Ejemplos: VMware ESXi, Microsoft Hyper-V Server, Xen.

🖥️ Tipo 2 (Hosted)

Se ejecuta como una aplicación sobre un sistema operativo ya instalado (Windows, Linux, macOS).

Ejemplos: VirtualBox, VMware Workstation.

Virtualización en el Centro de Datos

Imagine un servidor físico de gran potencia. Mediante virtualización, podemos aprovecharlo para tareas críticas aisladas:

  • VM Clientes: Para la base de datos de administración.
  • VM Web: Para alojar el servidor público de la empresa.
  • VM Laboratorio IA: Para testear modelos de Inteligencia Artificial sin afectar a los otros servicios.

Aunque comparten el mismo silicio, están aisladas: si una VM falla, las demás siguen operando con normalidad.

Servidores Físicos vs Virtualización

Característica Servidor físico tradicional Servidor virtualizado (VMs)
Uso del hardware Suele infrautilizarse Aprovechamiento óptimo con varias VMs
Aislamiento Depende del sistema operativo Alto: cada servicio en su propia VM
Flexibilidad Baja (alterar hardware es costoso) Alta (mover o clonar VMs es rápido)
Puesta en marcha Lenta (instalación de cero) Rápida (despliegue mediante plantillas)

Conceptos Clave

  • La virtualización optimiza el hardware compartiéndolo entre máquinas lógicas.
  • El hipervisor es la capa de software que gestiona los recursos.
  • La VM es un entorno completo con su propio sistema operativo.

Atención: El Coste de la Abstracción

La virtualización introduce sobrecarga (overhead). Como cada VM requiere ejecutar un sistema operativo completo, consume una parte importante de memoria y CPU solo para mantenerse activa. Por este motivo, la virtualización no es siempre la opción más ligera para desplegar cientos de microservicios.
