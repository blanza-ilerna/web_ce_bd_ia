Manipulación y Preparación de Datos

Del Caos a la Información Estructurada

El Paso Crítico: "Garbage In, Garbage Out"

Antes de que cualquier algoritmo de IA pueda aprender, los datos deben ser preparados meticulosamente. Los datos del mundo real suelen ser incompletos, inconsistentes y ruidosos. La fase de preparación de datos suele consumir entre el 60% y el 80% del tiempo de un proyecto de ciencia de datos.

Este proceso incluye la limpieza (tratamiento de valores nulos y duplicados), la transformación (convertir formatos, normalizar escalas) y la selección de características (elegir las variables más relevantes para el modelo).

A continuación, analizamos las herramientas fundamentales para esta etapa crítica, desde librerías de Python hasta soluciones Big Data y herramientas visuales.

El Ecosistema Python: Pandas y NumPy

Limpieza y Manipulación Ágil

Pandas y NumPy son el estándar de la industria para trabajar con datos en memoria. Ofrecen estructuras de datos flexibles y funciones vectorizadas de alto rendimiento para filtrar, limpiar y transformar datasets complejos.

Permiten detectar automáticamente valores faltantes (NaN), rellenarlos con estrategias estadísticas (media, mediana) o eliminar registros corruptos, así como realizar conversiones de tipos de datos de forma masiva.

Ejemplo de Limpieza con Pandas:

import pandas as pd
import numpy as np

# Crear DataFrame con datos "sucios"
data = {
    'nombre': ['Ana', 'Juan', 'Luis', np.nan],
    'edad': [25, 30, np.nan, 22],
    'salario': ['30000', '45000', '50000', 'missing']
}
df = pd.read_csv("datos_ventas.csv") # Simulación de carga

# 1. Tratar valores nulos (Imputación)
# Rellenar edad faltante con la media
df['edad'].fillna(df['edad'].mean(), inplace=True)

# 2. Conversión de formatos
# Convertir 'salario' a numérico, forzando errores a NaN
df['salario'] = pd.to_numeric(df['salario'], errors='coerce')

# 3. Eliminar filas que sigan teniendo nulos críticos
df.dropna(subset=['nombre', 'salario'], inplace=True)

print("Datos limpios:")
print(df.head())

Preprocesamiento para ML con Scikit-learn

Transformación y Normalización

Una vez los datos están limpios, deben transformarse para que los algoritmos matemáticos puedan procesarlos eficientemente. Scikit-learn ofrece módulos robustos como preprocessing para estas tareas.

StandardScaler / MinMaxScaler

Normalizan los datos numéricos para que todas las variables tengan la misma escala, evitando que una variable domine a las demás (ej. salario vs edad).

OneHotEncoder

Convierte variables categóricas (texto como "Rojo", "Verde") en vectores numéricos binarios que las redes neuronales pueden entender.

Pipline de Preprocesamiento:

from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.compose import ColumnTransformer
import pandas as pd

# Datos de ejemplo
X = pd.DataFrame({
    'edad': [25, 45, 35],
    'ciudad': ['Madrid', 'Barcelona', 'Madrid']
})

# Definir transformaciones
preprocessor = ColumnTransformer(
    transformers=[
        # Escalar 'edad' (media 0, desviación 1)
        ('num', StandardScaler(), ['edad']),
        # Codificar 'ciudad' como vectores binarios
        ('cat', OneHotEncoder(), ['ciudad'])
    ])

# Aplicar transformaciones
X_transformed = preprocessor.fit_transform(X)

print("Matriz procesada lista para entrenar:")
print(X_transformed)

Volúmenes Masivos: Apache Spark

Procesamiento Distribuido con PySpark

Cuando los datos superan la memoria de una sola máquina (Terabytes o Petabytes), Pandas no es suficiente. Apache Spark distribuye los datos a través de un clúster de servidores, permitiendo realizar las mismas operaciones de limpieza y transformación en paralelo.

Su módulo Spark MLlib incluye herramientas específicas para Feature Engineering a gran escala, como VectorAssembler y StringIndexer. 

from pyspark.sql import SparkSession
from pyspark.ml.feature import VectorAssembler, StringIndexer

spark = SparkSession.builder.appName("DataPrep").getOrCreate()

# Cargar dataset distribuido
df = spark.read.csv("hdfs://datos_log_masivos.csv", header=True, inferSchema=True)

# Indexar columna de texto (String -> Índice Numérico)
indexer = StringIndexer(inputCol="categoria", outputCol="categoria_index")
df_indexed = indexer.fit(df).transform(df)

# Ensamblar todas las características en un solo vector (requisito de Spark ML)
assembler = VectorAssembler(
    inputCols=["edad", "salario", "categoria_index"],
    outputCol="features")

output = assembler.transform(df_indexed)
output.select("features", "label").show(5)

Herramientas Visuales y No-Code

No siempre es necesario programar. Existen herramientas potentes para analistas que permiten preparar datos mediante interfaces visuales intuitivas.

Power BI

A través de su editor Power Query, permite realizar transformaciones complejas (ETL) paso a paso: filtrar filas, dividir columnas, reemplazar valores y combinar tablas, todo sin escribir código, aunque permite usar lenguaje M para lógica avanzada.

Google Cloud DataPrep

Servicio inteligente en la nube (basado en Trifacta) que sugiere transformaciones automáticamente al analizar los datos. Visualiza la distribución de valores en tiempo real, facilitando la detección de anomalías y outliers antes de exportar a BigQuery.
