El Encadenamiento Hash 🔗

Cómo se construye la arquitectura de confianza bloque a bloque.

Estructura de un Bloque

Para que la red sea inmutable, cada bloque de información debe estar vinculado matemáticamente con el anterior. En términos simplificados, cada bloque contiene:

📦 Transacciones

El conjunto de datos o acciones validadas (quién envió qué a quién).

🔑 Hash Propio

La huella digital única generada al procesar todo el contenido del bloque.

🔗 Hash Anterior

El valor hash del bloque que le precede, creando el vínculo de la cadena.

🧪 Demo en Vivo: Cadena de Bloques

¿Por qué es tan seguro? 🛡️

Si un atacante intenta modificar una sola transacción del Bloque #1:

  1. El contenido del Bloque #1 cambia.
  2. Automáticamente su Hash H1 se convierte en uno completamente diferente.
  3. El Bloque #2 registra el hash anterior original, que ya no coincide con el nuevo H1.
  4. La cadena queda rota y todos los bloques posteriores son inválidos.
  5. La red (con miles de nodos con la copia correcta) rechaza estas versiones manipuladas.

La contaminación se propaga hacia adelante

Para reescribir el historial de un bloque, un atacante tendría que recalcular también todos los bloques posteriores y superar el poder computacional del resto de la red. Con miles de nodos operando en paralelo, esto es en la práctica imposible.
